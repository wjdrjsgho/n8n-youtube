## **서론 및 중요성 강조**

- Claude 코드 학습 및 간단한 실습 시 하이쿠 모델을 사용하고, 복잡한 프로젝트나 실전 개발 시 소넷과 오푸스 모델을 활용하면 같은 Claude 구독 플랜으로도 훨씬 더 많은 작업을 수행할 수 있음.

- AI 네이티브 개발자가 되기 위한 중요한 내용인 토큰의 모든 것을 다룰 예정임.

- 이 영상은 다음 대상에게 유용함:

- Claude 코드를 쓰다가 토큰 제한이 빠르게 걸리는 사용자.

- 처음에는 AI 모델이 말을 잘 듣다가 조금만 사용하면 같은 코드를 반복해서 짜거나 AI 모델이 비효율적이 되는 것을 경험한 사용자.

- AI 모델을 효율적으로 사용하고 싶은 사용자.

## **토큰의 개념**

- 토큰은 AI가 데이터를 처리하는 최소 단위임.

- 인간이 글자나 단어 단위로 대화를 이해하는 것처럼 AI는 토큰 단위로 정보를 읽고 생성함.

- <b>예시</b>:

- 'Hello'는 약 1개 토큰을 차지함.

- 한국어 '안녕하세요'는 대략 5개 토큰을 차지함.

- AI 모델 사용 시 프롬프트를 입력하면 AI는 텍스트를 토큰 단위로 쪼개어 이해함.

- <b>토큰 크기의 비고정성</b>: 토큰의 크기는 고정되어 있지 않음. AI 모델마다 텍스트를 토큰으로 나누는 방식(토크나이저)이 다르기 때문에 같은 문장이라도 모델에 따라 토큰 수가 달라질 수 있음.

## **토큰의 중요성 및 컨텍스트 윈도우**

- AI 모델과의 모든 상호 작용(대화)은 모두 토큰으로 측정됨.

- <b>상호 작용의 두 가지 유형</b>:

- <b>입력(Input) 토큰</b>:

- 보내는 메시지, 첨부 파일 내용, 대화 히스토리, Claude 코드의 메모리 파일인 <code>claude.md</code> 파일 등 모든 것이 입력 토큰으로 계산됨.

- <b>출력(Output) 토큰</b>:

- Claude가 생성하는 응답, 코드, 설명, 분석 결과 등 Claude가 만들어내는 모든 것이 출력 토큰임.

- 입력 토큰은 Claude에게 주는 정보이고, 출력 토큰은 Claude가 주는 답변임.

- <b>컨텍스트 윈도우</b>:

- 한 번에 처리할 수 있는 최대 토큰 사용량임.

- 현재 Claude 모델은 20만 토큰의 컨텍스트 윈도우를 가지며, 이는 대략 책 한 권 분량에 해당함.

- 20만 토큰이라는 것은 입력과 출력을 합쳐서 최대 한 번에 20만 토큰까지 사용할 수 있다는 뜻임.

- <b>대화 진행에 따른 토큰 누적 방식</b>:

- 첫 번째 대화에서 메시지를 보내면 입력 토큰이 쌓이고, Claude가 답변하면 그 답변으로 인해 출력 토큰이 쌓임.

- 해당 대화에서 두 번째 메시지를 보낼 때, 두 번째 입력 토큰에는 두 번째 입력한 프롬프트뿐만 아니라 첫 번째 대화의 입력과 출력 토큰이 모두 포함됨.

- Claude가 또 답변하면 출력 토큰이 추가로 쌓임.

- 세 번째, 네 번째 대화로 계속 진행하면서 입력과 출력이 계속 누적되며, 이 모든 것이 20만 토큰이라는 컨텍스트 윈도우 안에서 관리됨.

- <b>컨텍스트 관리의 중요성</b>:

- 새로운 작업을 요청할 때는 반드시 <code>clear</code> 명령어를 통해 대화창에 있는 메시지 컨텍스트를 초기화하는 것이 매우 중요함.

- 컨텍스트 용량이 거의 찼는데 작업을 이어가고 싶다면 <code>compact</code> 명령어를 통해 메시지 컨텍스트를 압축하는 것도 중요함.

- Claude뿐만 아니라 AI 모델과 대화할 때 컨텍스트 윈도우를 관리하는 것이 매우 중요함.

- 간단한 질의를 할 때 이전 대화 내용이 가득 차 컨텍스트 윈도우를 많이 차지한 상태에서 프롬프트를 요청하면 토큰이 정말 빨리 소비됨.

- 토큰을 빠르게 소비하면 결국 Claude 코드의 사용량 제한에 걸려 더 이상 사용할 수 없는 상태가 됨.

## **Claude 모델별 토큰 및 가격 차이**

- <b>소넷(Sonnet 4.5) 모델</b>:

- 가장 균형 잡힌 모델로, 복잡한 업무나 어려운 코딩 작업에 최적화되어 있음.

- 가격: 입력 토큰 100만 개당 3달러, 출력 토큰은 15달러임.

- <b>하이쿠(Haiku) 모델</b>:

- 가장 빠르고 가벼운 모델임.

- 최첨단 지능을 유지하면서 응답 속도가 압도적으로 빠름.

- 간단한 질문이나 빠른 작업에 유리한 모델임.

- 가격: 입력 토큰 100만 개당 1달러, 출력 토큰은 5달러로 가장 저렴함.

- <b>오푸스(Opus 4.5) 모델</b>:

- 최고 성능의 프리미엄 모델임.

- 가장 뛰어난 지능과 실용성을 자랑하며, 매우 복잡하고 까다로운 작업을 처리할 때 사용하면 좋음.

- 다른 모델보다 가격이 다소 비쌈.

- 최근 발표된 이 모델은 복잡한 작업에는 더 빠르게 처리할 수 있어 결과적으로 소넷 4.5보다 토큰을 덜 소비하는 경향이 있음.

- <b>모델 선택의 중요성</b>: Claude 코드로 작업할 때 간단한 작업에는 하이쿠 모델, 복잡한 작업에는 소넷이나 오푸스 모델을 활용하면 토큰을 훨씬 효율적으로 사용할 수 있으므로, 작업에 맞는 모델을 선택해서 사용하는 것이 좋은 방법임.

- <b>소넷 100만 토큰 컨텍스트 윈도우 (베타)</b>:

- 현재 Claude API에서만 사용할 수 있으며, 추후 Claude 코드에서도 사용할 수 있게 된다면 더 많은 코드 맥락을 한 번에 기억하여 전체 프로젝트 구조를 더 정확하게 파악하고 훨씬 더 스마트하게 작업을 처리해 줄 것으로 기대됨.

## **Claude 코드 사용량 제한 및 확인 방법**

- <b>사용량 확인 방법</b>: Claude 코드에서 <code>usage</code> 명령어를 입력하면 현재 사용량을 즉시 확인할 수 있음.

- <code>usage</code> 명령어 입력 시 세 가지 유형으로 사용량이 표시됨:

- <b>Current Session (현재 세션)</b>:

- 5시간 단위로 리셋되는 사용량임.

- Claude 코드로 작업을 할 때 어떤 모델을 사용하든 Current Session의 한도에서 차감됨.

- 해당 세션이 모두 가득 차면 사용량 제한에 걸림.

- 제한이 몇 시에 리셋되는지 정확한 시간이 표시됨 (예: 오후 7시에 다시 사용할 수 있음).

- <b>Current Week: All Models (전체 모델 주간 제한)</b>:

- 일주일 단위로 리셋되는 제한임.

- 오푸스, 소넷, 하이쿠 모델을 사용했을 때 차감되는 주간 제한임.

- <b>Current Week: Sonnet Only (소넷 전용 주간 제한)</b>:

- 일주일 단위로 리셋되는 제한임.

- 소넷 모델을 사용했을 때 차감되는 주간 제한임.

- <b>모델별 차감 방식</b>:

- <b>오푸스 모델</b>: Current Session과 Current Week: All Models에서 차감됨.

- <b>소넷 모델</b>: Current Session, Current Week: All Models, 그리고 Current Week: Sonnet Only에서 모두 차감됨.

- <b>하이쿠 모델</b>: Current Session과 Current Week: All Models에서 차감됨.

- <b>소넷 모델의 이중 차감 이슈</b>: 소넷 모델이 'All Models' 주간 제한과 'Sonnet Only' 주간 제한 모두에서 차감되는 것에 대한 사용자들의 궁금증이 있음. 실제로 사용해 보면 소넷을 사용할 때 'All Models' 제한에서도 차감되고 있으며, 많은 개발자들이 'All Models'에서 소넷 차감을 제외해 달라는 의견을 남긴 것을 확인할 수 있음. 해당 이슈 관련해서는 추후 엔트로픽의 공식 발표가 있을 시 고정 댓글에 업데이트될 예정임.

## **토큰을 효율적으로 사용하는 방법**

- <b>1. 프롬프트 최적화</b>:

- '무엇 무엇을 만들어 줘'와 같이 범위를 제한하지 않고 프롬프트를 요청하면 Claude는 불필요한 기능이나 배포 설정, 테스트 코드 등 엄청나게 많은 코드를 생성하여 토큰을 폭발적으로 소모시킴.

- 간단한 구현만 원했는데 필요하지 않은 것까지 모두 만들어서 토큰이 과도하게 소모될 수 있음.

- 프롬프트를 요청할 때는 명확하게 범위를 제한하여 요청한다면 토큰 사용량이 훨씬 줄어들 것임.

- 핵심 키워드는 범위를 제한할 수 있는 표현들임.

- <b>2. 하이쿠 모델 적극 활용</b>:

- 간단한 작업이나 학습을 할 때도 오푸스나 소넷 모델만 사용하는 경우가 종종 있음.

- 하이쿠 모델은 속도도 매우 빠르고 토큰 효율도 최고로 좋음.

- Claude 코드를 학습할 때나 간단한 실습을 할 때는 하이쿠 모델을 사용하고, 복잡한 프로젝트나 실전 개발을 할 때는 소넷과 오푸스 모델을 활용하면 같은 Claude 구독 플랜으로도 훨씬 더 많은 작업을 할 수 있음.

- <b>3. 컨텍스트 관리</b>:

- 새로운 작업을 할 때는 반드시 <code>clear</code> 명령어를 사용하여 불필요한 이전 대화 내용을 초기화해야 함.

- 컨텍스트 윈도우는 제한되어 있고, 불필요한 대화가 쌓인 상태에서 다시 프롬프트를 요청하게 된다면 누적된 메시지 컨텍스트가 입력 토큰으로 소모되어 많은 토큰을 사용하게 됨.

- 이러한 방식은 공식 문서에서도 권장하는 내용임.

- 작업을 계속해서 이어나가고 싶은데 컨텍스트가 거의 찼다면 <code>compact</code> 명령어를 이용해서 메시지를 압축하여 공간을 확보할 수 있음.

- <b>4. 컨텍스트 윈도우를 차지하는 불필요한 요소 줄이기</b>:

- 개발 편의성을 위해 서브 에이전트, MCP 등 Claude 코드의 확장 기능을 과도하게 설치하면 이러한 것들도 결국 컨텍스트 윈도우를 차지하는 요소가 됨.

- Claude 코드의 메모리 파일인 <code>claude.md</code> 파일도 길게 작성하면 컨텍스트 윈도우를 차지하는 요소가 되어 토큰을 빨리 소모시킴.

- 컨텍스트 윈도우가 가득 찬 상태에서 개발을 진행하면 Claude 코드가 이미 짠 코드를 다시 개발하는 등 비효율적이 되거나, 성능이 저하될 확률이 올라감.

- 개발을 진행할 때에는 서브 에이전트, MCP, 커스텀 커맨드 등 확장 기능들을 정말 필요한 것만 엄선하여 설치하고, 사용하지 않는 것은 과감하게 제거해야 함.

- <b>토큰 효율적 사용법 정리</b>:

- 명확한 프롬프트로 범위를 제한함.

- 작업에 맞는 적절한 모델을 선택함.

- 컨텍스트를 깔끔하게 관리함으로써 같은 요금제를 사용하더라도 훨씬 더 많은 작업을 할 수 있음.

## **AI 네이티브 개발자의 핵심 본질: 컨텍스트 싸움**

- AI 개발을 워크플로우에 잘 녹이는 AI 네이티브 개발자가 되기 위해서는 결국 컨텍스트 싸움이라는 본질을 이해해야 함.

- 어떤 AI 모델을 사용하든 결국 한 번의 작업을 처리할 수 있는 토큰의 크기인 컨텍스트 윈도우가 정해져 있을 것임.

- 이러한 컨텍스트 윈도우 안에서 꼭 필요한 컨텍스트만 정확하게 전달해야 함.

- 앞으로 AI 모델이 더 똑똑해지고 컨텍스트 윈도우가 더 커지더라도, 결국 해야 할 일은 컨텍스트를 효과적으로 관리하는 것임.

## **Claude 코드 강의 안내**

- Claude 코드의 스펙을 제대로 학습하고 이러한 스펙을 개발 워크플로우에 제대로 녹이고 싶은 사람들을 위해 '클로드코드 완벽 마스터 강의'를 론칭함.

- 해당 강의에서는 다음 내용을 다룸:

- 터미널 기초부터 시작하여 커스텀 커맨드, 서브 에이전트, 훅.

- 최근 업데이트된 Claude 스킬, 플러그인, 마켓플레이스 등 Claude 코드의 모든 스펙.

- 이러한 스펙을 공식 문서에서 권장하는 AI 개발 워크플로우에 잘 활용할 수 있도록 반복적으로 설명함.

- 최근 수강 후기를 보면 원리를 반복적으로 설명하는 것에 대한 만족도가 높음.

- 초보자도 익숙해질 수 있도록 친절하게 설명하며, 관심 있는 사람들을 위해 고정 댓글에 할인 링크를 기재함.
