## **AI 직원을 온보딩하고 활용하는 방법: Mixmees Agent를 중심으로**

- AI 직원은 오전 10시에 메일과 캘린더 정리, 주식 시장 변화 분석, 영상 제작 아이디어 추천, 최근 트렌딩 AI 소식 요약 등 다양한 작업을 수행함.

- Mixmees Agent를 설치하면 이러한 작업들을 직원들에게 손쉽게 위임하고 보고를 받아볼 수 있음.

- Mixmees Agent 설치 및 기본 셋업 방법에 대한 지난 영상에 이어, 이번 영상에서는 AI를 단순히 요청-답변 도구가 아닌, 매일 정해진 시간에 출근하여 업무 매뉴얼대로 일하고 보고하는 'AI 직원'으로 키우는 방법을 다룸.

- AI를 직원처럼 활용하려면 기본적으로 셋업해야 할 다섯 가지 영역이 있음.

- 이 영상은 Mixmees Agent 기본 설치와 Telegram, Slack 연결이 완료된 상태를 가정하고 진행함.

## **AI 직원 온보딩을 위한 5가지 핵심 영역**

- AI 직원을 활용하려면 다음 다섯 가지 영역을 설정하는 것이 중요함.

- <strong>직원 아이덴티티 (`soul.md`) 작성</strong>: 직무 기술을 포함한 직원의 정체성 설정 파일.

- <strong>기억력 관리 (`user.md`, `memory.md`) 최적화</strong>: 피드백이나 회사 룰 등을 기억할 수 있도록 설정.

- <strong>업무 매뉴얼 (`agent.md` 및 스킬) 세팅</strong>: 특정 업무 진행 시 따를 매뉴얼 설정.

- <strong>업무 스케줄 (`cron`) 셋업</strong>: 정해진 시간에 특정 업무를 수행하도록 설정.

- <strong>보고 채널 설정</strong>: 업무 결과를 보고받을 채널 설정.

- 각 영역별 세부 설정 방법과 운영 팁들을 다룸.

## **1. 직원 아이덴티티 설정 (`soul.md`)**

- `soul.md` 파일은 직원의 아이덴티티를 알려주는 문서로, Hermis Agent가 세션을 시작할 때 가장 먼저 읽는 파일임.

- 이 파일을 잘 고도화해야 원하는 유형의 직원을 만들 수 있음.

- 직원이 어떤 말투로 답할지, 어떤 성격과 태도로 일할지, 어떤 기준을 지켜야 하고 어떤 작업을 하면 안 되는지 등 기본적인 정체성을 설정함.

- <strong>`soul.md` 작성 팁</strong>

- 파일을 직접 타이핑하기보다, Hermis Agent와 함께 초안을 만들고 피드백을 주며 고도화하는 것을 추천함.

- <strong>초안 생성 프롬프트 예시</strong>: '너는 내 개인 비서 AI고 이름은 자비스고, 다음 책임 범위로 `soul.md` 초안을 만들어 줘'라고 요청하며, Identity, Mission, Operating Context Pointer, Core Tone, Boundary, Reporting Scheme 같은 섹션을 제시함.

- 각 섹션에 들어갈 내용을 구체적으로 요청하면 더 상황에 맞게 작성해 줌.

- <strong>`Operating Context Pointer`의 중요성</strong>

- `agent.md`는 특정 프로젝트 폴더에 지정되거나 디폴트 프로젝트가 있을 때만 로드됨.

- Telegram DM 등으로 여러 작업을 요청할 경우, 특정 프로젝트의 `agent.md`만 로드되는 것은 비효율적일 수 있음.

- 직원이 항상 지켜야 하는 전반적인 업무 프로세스(예: 커뮤니케이션 방식, 참고 파일 등)에 대해 별도의 `agent.md` 문서를 만들고, `soul.md`에서 이 파일을 포인터로 연결하여 필요할 때 로드하도록 설정하는 것이 좋음. (예: `opt/data/agent.md`)

- `soul.md`에는 직원의 아이덴티티 기준으로만 내용을 넣고, 업무의 구체적인 프로세스는 `agent.md` 문서로 별도로 분리하는 것을 추천함.

## **2. 기억력 관리 설정 (`user.md`, `memory.md`)**

- Hermis는 메모리 관리를 어느 정도 자동으로 수행하며, `user.md`와 `memory.md` 파일을 자동으로 생성하고 관리함.

- <strong>메모리 관리 유의사항</strong>

- <strong>하드 리밋 존재</strong>: `memory.md`는 약 2,200자, `user.md`는 약 1,375자 정도의 글자 수 제한이 있음.

- 무한정 기억 요청 시 파일이 금방 차오를 수 있으므로, 하드 리밋을 인지하고 관리해야 함. 이는 AI도 모든 사소한 디테일을 항상 기억하기 어렵다는 실제 직원과 비슷한 맥락임.

- <strong>세션 시작 시 주입</strong>: `user.md`와 `memory.md`는 세션을 시작할 때 한 번 주입됨.

- 세션 중 특정 사항을 추가로 기억하게 시킨 후 즉시 반영을 원하면, `/new` 등으로 새 세션을 열어 다시 로드시켜야 함.

- <strong>`user.md`, `memory.md` 작성 팁</strong>

- 기본 설정은 영문으로 해두는 것을 추천함. 영문 토큰이 한글보다 효율적이라 더 많은 내용을 담을 수 있음.

- 한국어 뉘앙스가 중요한 표현(예: 회사명, 호칭)만 한국어로 넣어 줌.

- 내용이 길어질 경우 완벽한 문장 대신 구문이나 키워드 중심으로 압축하여 작성하는 것을 추천함.

- <strong>메모리 추가 요청 예시</strong>: '테이블 사용을 자주 해 줬으면 좋겠고 결론부터 항상 얘기하고 회사는 구시 컴퍼니 호칭은 대표님으로 해 달라'고 요청하면 `user.md`에 해당 내용이 추가됨.

- <strong>기억 검진 및 관리</strong>

- 최소 한 달에 한 번, 또는 주간/격주로 기억 검진을 돌려주는 것을 추천함.

- 프롬프트: '지금 내 `user.md` `memory.md`를 점검해 주고, 지우지는 말고 정리하면 좋은 거 후보로 추천해 줘'라고 요청함.

- 제안된 정리 후보를 보고 선별하여 내용을 줄여나감.

- <strong>자동화 팁</strong>: 매월 1일 오전 11시에 점검하고 섹션 테이블로 보고하도록 크론을 셋업하면 자동으로 기억 검진 리포트를 받아볼 수 있음.

## **3. 업무 매뉴얼 설정 (`agent.md` 및 스킬)**

- `agent.md` 문서에는 조직에서 일할 때 지켜야 할 공통 규칙(예: 보안, 판단, 작업 경로, 보고 양식 등)을 작성함. 이는 직원의 커스텀에 맞게 설정됨.

- <strong>`agent.md` 작성 팁</strong>

- `agent.md` 내용이 방대해지는 것을 방지하기 위해 '작업 경로'와 같은 섹션을 만들어 인덱스처럼 활용하고, 구체적인 가이드라인(예: 커뮤니케이션 규칙)은 별도 파일로 만들고 포인터로 연결하는 것이 좋음.

- <strong>스킬스 활용</strong>

- `agent.md`가 공통 규칙이라면, 직무를 수행하면서 구체적인 작업을 할 때 따라야 하는 업무 매뉴얼은 '스킬'로 별도 제작하고 활용하도록 세팅함.

- Hermis는 스킬을 자동으로 생성해 주는 기능이 있어, 직원이 어떤 목적으로 어떤 결과물을 원하는지 설명만 잘하면 알아서 스킬을 만들어 줌.

- <strong>모닝 브리프 스킬 생성 예시</strong>

- 세션을 새로 시작한 후, '모닝 브리프를 작성해 줘. 매뉴얼은 다음 단계로 구성해 줘: Gmail에서 최신 메일 3개 가져와 요약, Google 캘린더에서 오늘 일정 시간순으로 정리, 최근 1주일간 비개발자나 SB 업체에게 도움 될 만한 AI 뉴스 3건 리서치, 1~3을 묶어서 한 페이지짜리 모닝 브리프 리포트로 정리. 보고 양식은 표 위주, 강조색은 오렌지, 위험은 빨강으로 설정해 줘.'라고 요청함.

- Gmail 및 Google 캘린더 연동이 필요한 경우, 연동 방법을 안내받아 설정한 후 다시 요청하면 브리프를 생성해 줌.

- PDF 형태로 브리프를 다운로드 받을 수 있음.

## **4. 업무 스케줄 설정 (`cron`)**

- 업무 매뉴얼까지 인수인계했다면, 일정 시간에 업무를 수행할 수 있도록 업무 스케줄을 셋업해야 함.

- Slack 비즈니스 채널에 모닝 브리핑을 평일 오전 10시에 자동으로 업로드해 달라고 요청함.

- <strong>크론 스케줄링 옵션 5가지 패턴</strong>

- <strong>Deliver Local</strong>: 업데이트만 필요하고 확인이 불필요한 작업에 사용. 파일만 저장하고 종료됨.

- <strong>Silent</strong>: 결과물 내용이 없을 경우 알람을 보내지 않음 (출력 결과에 `[Silent]`가 있으면 알람 없음).

- <strong>Watchdog</strong>: 모델 호출 없이 스크립트만 실행하여 크론을 실행함. 토큰 비용 절감 용도로 유용함.

- <strong>Context From</strong>: 이전에 돌린 크론의 결과물을 받아 다음 크론에서 활용할 수 있음 (크론 체이닝).

- 주의할 점은, 이전 크론의 결과물을 항상 기다리는 것이 아니라, 다음 크론이 실행되는 시점에 최신 결과물을 가져오는 방식이므로 시간대 설정에 유의해야 함.

- <strong>주간 종합 보고</strong>: 데일리 작업 결과물들을 위클리 단위로 받아 특정 작업을 수행하고 요약 보고하는 방식으로 활용할 수 있음.

- 이러한 옵션들을 구체적으로 기억할 필요 없이, 원하는 업무 형태를 설명하면 Hermis Agent가 알아서 옵션에 맞게 설정해 줌.

- 더 자세한 옵션은 공식 다큐멘테이션을 참고함.

## **5. 보고 채널 설정**

- AI 직원이 업무를 완료하면 지정된 채널에 보고하도록 설정함.

- Slack 비즈니스 채널로 모닝 브리핑 결과가 전달되도록 설정함 (위 업무 스케줄 설정과 연계).

## **AI 직원 관리 및 운영 팁**

- <strong>Slack 설정 문제 해결</strong>

- <strong>채널에 앱 추가</strong>: Slack 채널의 '통합'에서 앱을 선택하여 자비스(AI 직원)를 추가해야 함.

- <strong>멘션하여 요청</strong>: 기본적으로 `@자비스 [요청 내용]`처럼 멘션해야 답변함.

- <strong>권한 설정</strong>: Slack 앱 설정에서 봇 토큰 범위(Bot Token Scopes)와 이벤트 구독(Event Subscriptions)에 필요한 권한(예: `channels:write`, `im:write`)이 모두 설정되어 있어야 함. (Slack Manifest를 통해 설치 시 한 번에 설정 가능)

- <strong>권한 변경 후 리인스톨</strong>: 권한 설정을 변경한 후에는 반드시 'Install App' 옵션에서 'Reinstall'을 눌러 Slack 워크스페이스에 반영해야 적용됨.

- <strong>멘션 없이 답변 설정</strong>: 특정 채널에서는 멘션 없이도 AI가 답변하게 하고 싶다면, `data/config.yaml` 파일의 `slack` 섹션 내 `pre_response_channels`에 해당 채널 ID를 추가함.

- <strong>변경 사항 반영</strong>: 설정 변경 후 게이트웨이 리스타트가 필요하며, 도커 컨테이너 자체를 리스타트하는 것이 더 안정적임 (호스팅어의 도커 매니저에서 리스타트 등).

- <strong>Slack에서 슬래시 커맨드 활용</strong>

- Telegram에서는 `/restart`, `/compress` 등 Mixmees 제공 슬래시 커맨드 사용 가능함.

- Slack은 슬랙 자체 커맨드와 겹쳐 사용 불가함.

- <strong>해결책</strong>: '리스타트' 단독 메시지를 `/restart` 명령으로, '컴프레스' 단독 메시지를 `/compress` 명령으로 변환하는 플러그인을 생성 요청함. 플러그인 생성 후 재시작하면 Slack에서도 유사한 방식으로 커맨드를 활용할 수 있음. (도커 컨테이너 자체 리스타트 권장)

- <strong>툴콜 로그 가독성 개선</strong>

- Slack 스레드에서 툴콜(Tool Call)이 너무 많이 달리면 가독성이 떨어짐.

- 요청: '로그 가독성 개선을 적용해 줘. 툴콜을 기본적으로 하되, 'Still working' 같은 반복성 메시지는 숨겨 줘'와 같이 요청하여 툴콜을 압축시킬 수 있음.

- 설정 키 예시를 활용하여 본인의 선호도에 맞게 조정 가능함.

- <strong>권한 승인 요청 줄이기 (`Approvals Mode`)</strong>

- 복잡하거나 민감한 작업 시 AI가 권한 승인 요청을 자주 할 수 있음.

- 해결책: 프로필의 `Approvals Mode`를 `smart`로 변경 요청함.

- `manual` 모드는 자주 체크하며 승인 요청, `smart` 모드는 안전한 명령은 자동 실행하고 위험한 명령만 승인 요청(추천), `off` 모드는 거의 모든 것을 자동으로 실행함.

- <strong>결과물 첨부 파일 자동화</strong>

- PDF, 이미지 등 결과물이 자동으로 첨부되지 않는 경우가 발생할 수 있음.

- 요청: '웬만한 파일 결과물은 항상 Slack이나 Telegram에 붙여서 보고해 줘'와 같이 리포팅 규칙을 설정하여 첨부 파일을 자동으로 받을 수 있게 함.

- VPS 등 서버에 파일이 저장되는 경우, 메신저에서 바로 다운로드하여 활용하기에 유용함.

- <strong>코덱스 연결 이슈 해결 (최근 업데이트)</strong>

- OpenAI Codex 구독 계정 연동 시 백엔드 코드 이슈로 불안정성이 발생한 사례가 있음.

- Mixmees 업데이트를 통해 해결될 수 있으나, 업데이트 후에도 문제가 지속될 경우 제공되는 패치 코드(터미널 명령어)를 활용하여 수동으로 해결할 수 있음. (도커 컨테이너 이름에 맞게 수정하여 적용)

- 구독 계정을 연동하는 방식은 API 방식보다 가성비가 좋으나, 어느 정도 불안정성을 감수해야 함.

## **결론 및 향후 전망**

- AI 직원을 만드는 과정은 신입 사원을 온보딩하는 것과 비슷함. 직원의 아이덴티티, 기억력, 업무 매뉴얼, 스케줄, 보고 채널 등 5가지 요소를 챙겨야 함.

- 처음부터 모든 것을 갖출 필요는 없으며, `soul.md`와 `user.md` 설정 후 간단한 업무부터 시작하여 피드백을 통해 개선하고, 스케줄 및 보고 채널을 설정하여 복잡한 업무를 점차 추가해 나가는 것을 추천함.

- AI 직원을 통해 원하는 결과물을 하나 만들어 보고, 그것을 지속적으로 받아볼 수 있도록 자동화 설정을 하면 됨.

- 앞으로 1~2년 사이에 직원형 AI 운영 패턴이 대세가 될 것으로 예상됨. 지금 시점에서 운영 감각을 익혀두면, 향후 더 좋은 에이전트 툴이 나오더라도 기본적인 시스템 구조를 바탕으로 기존 문서를 가져다 활용할 수 있음.

- 사업 확장이나 더 많은 업무 위임 시에는 여러 프로필(직원)을 추가하여 콘텐츠, 영업, 리서치 등 담당 영역을 나누어 활용하는 방법도 있음. (이러한 내용은 추후 영상에서 다룰 예정)
