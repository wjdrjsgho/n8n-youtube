## **AI 비서의 필요성 및 Hermis Agent 소개**

- AI가 개인 비서처럼 스스로 업무를 진행하고 알림을 해 주는 방식에 대한 필요성을 언급함.

- 
        자동화된 AI 비서의 예상 역할:

- 매일 오전 10시, 사용자의 일정, 메일 내용, 업계 뉴스 등을 포함한 모닝 브리핑 제공.

- 투자 중인 주식 관련 소식 정리 및 보고.

- 자동화 세팅 방법이 간단함을 강조함: 원하는 자동화를 말로 설명만 하면 AI가 직접 세팅함.

- 오늘 만들 AI 비서는 시키지 않아도 알아서 일하는 똑똑한 AI 비서임.

- 
        기존 도구(클로드 코드, 코덱스)로도 자동화는 가능하나, Hermis Agent를 추천하는 이유:

- AI를 비서처럼 만들고 24시간 켜두며 사용자의 선호도와 반복 업무를 기억하고, 점차 성장하는 직원으로 만들고 싶을 때 더 적합함.

- Hermis Agent를 활용하여 나만의 AI 비서를 새로 구축하고, 비서를 성장시키는 구조를 직접 만드는 방법을 안내할 예정임.

## **AI 도구 개념 및 Hermis Agent와 기존 도구의 차이점**

- 
        <strong>클로드 코드, 코덱스, 커서 같은 기존 강력한 AI 도구의 활용:</strong>

- 엄밀히 따지면 클로드 코드나 코덱스로도 비서 같은 직원을 만들 수 있음.

- 예시: 역할 프롬프트, 에이전트별 페르소나 부여, 운영 기준 문서 생성, 크론 또는 내장 자동화 기능 활용, 슬랙/텔레그램 등 채널 직접 연결.

- 이러한 기능을 통해 클로드 코드나 코덱스를 직원처럼 활용할 수 있음.

- 
        <strong>클로드 코드 및 코덱스의 한계점:</strong>

- <strong>기본 출발점의 차이:</strong> 기본적으로 프로젝트를 열고 그 자리에서 작업을 맡기는 '작업 도구'에 가까움.

- <strong>24시간 비서로 사용 시 고려사항 증가:</strong>

- 애초에 설계된 UI/UX와 다르게 커스텀해야 할 사항이 많음.

- 개발자에게는 흥미로운 프로젝트일 수 있으나, 비개발자에게는 직접 설정해야 하는 부담이 큼.

- 최적화된 솔루션을 만드는 데 한계가 있을 수 있음.

- <strong>모델 선택의 자유도 부재:</strong>

- 클로드 코드는 기본적으로 Anthropic 모델 중심으로만 활용 가능함.

- 코덱스는 OpenAI 모델만 활용 가능함.

- <strong>수동 설정의 비효율성:</strong>

- 여러 작업을 할 때 특정 작업은 코덱스, 다른 작업은 클로드 코드에게 시키고 싶을 수 있음.

- 24시간 돌아가는 비서에게 매번 수동으로 모델 설정을 해줘야 하면 매우 번거로움.

- 
        <strong>Hermis Agent의 장점:</strong>

- <strong>탄생 목적:</strong> 처음부터 24시간 비서가 되기 위해 설계된 프로젝트임.

- <strong>채널 연동 용이성:</strong> 텔레그램, 슬랙, 디스코드 등 다양한 채널과 손쉽게 연동 가능함.

- <strong>강력한 메모리 기능:</strong>

- 사용자가 요청한 지시사항이나 개인 정보를 잘 기억하고 꾸준히 활용함.

- <strong>반복 업무 자동화:</strong> 크론을 활용하여 반복 업무를 손쉽게 예약하고 실행할 수 있음.

- <strong>자동 스킬 생성 및 업데이트:</strong>

- 자주 하는 업무에 대해 자동으로 스킬을 생성하고 업데이트함.

- 비개발자 입장에서 스킬을 손쉽게 생성하고 활용할 수 있다는 장점이 있음.

- <strong>AI 개인 비서 운영에 초점:</strong> 이러한 기본 기능들을 탑재하고 있어 AI를 개인 비서처럼 운영하는 구조에 더욱 적합함.

- <strong>오픈소스 프로젝트 및 모델 자유도:</strong>

- 오픈소스 프로젝트이므로 특정 모델에 종속되지 않음.

- 다양한 모델을 테스트하고 유즈케이스에 맞게 동시에 활용할 수 있음.

- 예시: 글쓰기 작업은 Opus 4.7, 코딩 작업은 Codexus GPT 5.5 등 분할 활용 가능.

## **AI 도구 선택 멘탈 모델 제시**

- 너무 많은 AI 도구 속에서 어떤 툴을 언제 사용해야 할지 고민될 때 참고할 멘탈 모델을 제안함.

- 
        <strong>클로드 코드/코덱스 활용 시점:</strong>

- 책상에 앉아 직접 어떤 실무를 할 때 도움을 받고 싶을 때.

- 
        <strong>Hermis Agent 활용 시점:</strong>

- 책상에 있지 않더라도 개인 비서나 담당 직원들이 각자의 역할에 맞게 어떤 작업을 반복적으로 실행해 주기를 원할 때.

- <strong>툴 구분의 유연성:</strong> 처음부터 툴 구분을 명확히 할 필요는 없음.

- <strong>작업 이전 방법:</strong> 클로드 코드나 코덱스로 작업을 하다가 '이것은 정형화된 반복 업무로 만들 수 있겠다'는 생각이 들면, Hermis Agent 같은 에이전트로 작업을 이전시키는 것도 좋은 방법임.

## **Hermis Agent 설치 방법 (VPS 활용)**

- 
        <strong>Hermis Agent 설치 방식:</strong>

- <strong>전용 하드웨어 활용:</strong> 맥 미니 같은 컴퓨터를 구매하여 24시간 켜놓고 활용하는 방법이 있음. 민감한 파일을 내 장비 안에서만 두고 활용하고 싶을 때 적합함.

- <strong>VPS 활용:</strong> 호스팅어 같은 서비스에서 VPS(Virtual Private Server)를 구매하여 웹 클라우드 서버를 활용해 에이전트를 설치하고 24시간 동안 활용하는 방법이 있음. 별도의 컴퓨터 구매가 부담스럽거나 쉽게 시작하고 테스트해보고 싶을 때 좋은 선택지임.

- 
        <strong>VPS 서비스 선택 (호스팅어 추천):</strong>

- 설정 과정이 간단하고 가격이 저렴한 호스팅어 서비스를 활용함.

- 추가 할인 링크가 제공될 예정임.

- 호스팅어 웹사이트(`hostinger.com/pricing`)에서 VPS 호스팅을 선택함.

- 월 13,000원 정도에 구매 가능 (구독 기간에 따라 가격 변동).

- 'KVM2' 플랜을 선택하고 바로 구매하기를 누름.

- 
        <strong>호스팅어 VPS 구매 및 설정 과정:</strong>

- <strong>서버 위치 선택:</strong> 아시아 지역 (싱가포르, 말레이시아, 인도네시아 등) 중 하나를 선택함.

- <strong>운영 체제 선택:</strong> Ubuntu 2404 LTS를 선택함.

- <strong>결제 진행:</strong> 결제 시 '멀웨어 스캐너'와 '도커 매니저' 설치 옵션이 뜨면 둘 다 체크하여 설치함.

- 결제 완료 후 오버뷰 화면이 뜨면 왼쪽 메뉴에서 '도커 매니저'를 선택함.

- 
        <strong>Hermis Agent 컨테이너 배포:</strong>

- 도커 매니저에서 '새로운 컨테이너 생성' 또는 '원클릭 디플로이'를 누름.

- '치 템플릿'에서 'Hermis Agent'를 찾아 선택함.

- 어드민 유저네임은 'Hermis' 그대로 사용하고, 패스워드는 별도로 복사해 둠 (예: 데모 비디오용으로 유저네임 수정).

- 이렇게 설정하면 동일한 VPS 컴퓨터 안에서 여러 컨테이너를 동시에 추가하고 활용할 수 있음.

- 기본 설치가 완료됨.

## **Hermis Agent 초기 설정 (모델 및 메시징 플랫폼 연동)**

- 
        <strong>컨테이너 터미널 접속:</strong>

- 생성된 컨테이너에 있는 '터미널' 버튼을 눌러 터미널을 염.

- Hermis Agent가 설치된 컨테이너로 진입함.

- 
        <strong>Hermis Agent 셋업 시작:</strong>

- 터미널에 `hermes setup` 명령어를 입력함.

- 이미 Hermis Agent가 설치되어 있으므로 바로 셋업 옵션이 나타남.

- 'Quick Setup'을 선택하고 Enter를 누름.

- 
        <strong>모델 프로바이더 설정:</strong>

- 처음 시작할 때는 Codexus를 추천함 (ChatGPT 구독 계정으로 그대로 활용 가능하기 때문).

- Codexus를 선택하고 Enter를 누름.

- 제공된 링크를 눌러 들어가 코드를 복사하여 터미널에 넣어줌.

- 모델은 GPT 5.5를 선택함.

- 터미널 설정은 컨테이너 안에서의 실행이므로 'local'을 선택함.

- 
        <strong>메시징 플랫폼 설정 (텔레그램 및 슬랙):</strong>

- 메시징 플랫폼 설정 옵션에서 텔레그램과 슬랙을 선택함 (커서 이동이 안 될 경우, 숫자 1(텔레그램)과 3(슬랙)을 차례로 입력하고 Enter).

- 두 플랫폼 선택 후 Enter를 누르면 텔레그램부터 설정을 진행함.

- 
        <strong>텔레그램 설정:</strong>

- BotFather에서 봇을 생성해야 함.

- 텔레그램 검색창에 '@BotFather'를 검색하여 접속함.

- `/newbot` 명령어를 입력하고 봇 이름(예: 자비스)과 영문 봇 유저네임(예: 자비스봇)을 설정함.

- 생성된 봇 토큰을 복사하여 터미널에 넣어줌.

- 봇이 불특정 다수에게 답변하는 것을 방지하기 위해 사용자 ID를 입력해야 함.

- 텔레그램 검색창에 '@userinfobot'을 검색하여 접속, 본인 ID를 확인하고 복사하여 터미널에 넣어줌 (여러 ID를 콤마로 구분하여 입력 가능).

- 본인 사용자 ID를 '홈'으로 설정함. 텔레그램 설정이 완료됨.

- 
        <strong>슬랙 설정:</strong>

- `api.slack.com/app`에 접속하여 앱을 생성해야 함.

- 'Create New App' > 'From an app manifest'를 선택함.

- 워크스페이스를 선택하고 다음을 누른 후 앱을 생성함.

- 앱 설정 화면에서 'Socket Mode'에 들어가 'Enable Socket Mode'를 활성화함.

- 토큰 네임(예: 자비스 토큰)을 설정하고 `connections:write` 스코프가 들어있는 상태에서 토큰을 생성하고 복사하여 메모장에 임시로 붙여넣어 둠. (앱 토큰)

- <strong>권한 설정 (Manifest JSON 활용):</strong>

- 새로운 터미널을 열고 `cat opt/data/slack_manifest.json` 명령어를 입력하여 Hermis Agent가 제공하는 슬랙 매니페스트 JSON 내용을 확인함.

- Slack 앱 설정 페이지의 'App Manifest' 섹션에 해당 JSON 내용을 붙여넣고 'Display Information' 등을 수정 후 변경사항을 저장함.

- 이 과정으로 원래 수동으로 설정해야 하는 권한들과 이벤트 구독 권한이 한 번에 설정됨.

- <strong>앱 설치:</strong> 'Install App to Workspace'를 클릭하여 생성한 봇을 본인의 슬랙 채널에 설치함.

- 설치 완료 후 발급되는 'Bot User OAuth Token'(`xoxb`로 시작)을 복사하여 터미널에 넣어줌.

- 이전에 임시 저장해 둔 'App Token'(`xapp`로 시작)도 복사하여 터미널에 넣어줌.

- 슬랙 유저 ID를 입력해야 함: 슬랙 프로필에서 '멤버 ID 복사'를 한 후 터미널에 넣어줌.

- 메인이 될 슬랙 채널을 선정하고 해당 채널의 ID를 복사하여 '홈 채널'로 설정함. 슬랙 설정이 완료됨.

- 
        <strong>게이트웨이 설정 및 연동 테스트:</strong>

- 현재 접속 중인 터미널에서 `hermes`를 입력하여 Hermis Agent 터미널 버전을 실행함.

- '텔레그램과 슬랙 연결을 테스트해 달라'고 요청하여 게이트웨이 설정을 진행함. (해당 프롬프트는 상세 설명 가이드에 제공될 예정임.)

- 슬랙에서 봇을 사용하려면 해당 채널에 앱을 추가해야 함: 홈 채널에 들어가 '통합' > '앱 추가'에서 '자비스' 봇을 추가함.

- 연결이 완료되었다는 메시지가 뜨면, 텔레그램 봇 링크로 들어가 봇에게 메시지를 보내 테스트함. ('잘 들립니다. 무엇을 도와드릴까요?' 같은 답변 확인)

- 슬랙에서도 `@자비스`를 멘션하여 메시지를 보내 테스트함. ('안녕하세요. 무엇을 도와드릴까요?' 같은 답변 확인)

- 텔레그램과 슬랙 연동이 성공적으로 완료됨.

## **Hermis Agent 파일 구조 이해 및 VS Code 연동**

- 
        <strong>Hermis Agent 파일 저장 위치 확인:</strong>

- 컨테이너 관리 페이지에서 생성한 컨테이너의 '매니지'를 누름.

- 'YAML 에디터'를 클릭하여 설정된 볼륨 맵핑을 확인.

- `. /data`와 `/opt/data`로 볼륨이 설정되어 있음:

- `/opt/data`는 도커 컨테이너 내부에서 데이터가 저장되는 경로를 의미함.

- `/data`는 도커 컨테이너 외부, 즉 VPS 호스트 볼륨에서 동일한 데이터를 싱크하여 저장하는 폴더를 의미함.

- `/data` 폴더 안에는 `config`, `skills` 등 Hermis Agent의 기본 파일들이 저장됨.

- 컨테이너 내부 터미널이 아닌, 우측 상단의 '터미널' 버튼을 눌러 VPS의 루트 경로에서 터미널을 열 수 있음.

- 루트 터미널에서 `cd /docker` 후 `cd hermes-agent-...` (생성된 컨테이너 이름)으로 들어가면 `data` 폴더를 확인할 수 있음. 이 `data` 폴더가 `/` 슬래시 데이터와 동일한 `persistent data` (영구 데이터)임.

- <strong>결론:</strong> YAML 에디터에서 앞에 부분의 `/data`는 컨테이너 외부의 저장 경로를, 뒤에 부분의 `/opt/data`는 컨테이너 내부의 저장 경로를 의미하며, 이 설정이 연구적으로 파일이 저장되도록 함.

- 
        <strong>추가 작업 폴더 (워크스페이스) 맵핑:</strong>

- 생성하는 결과물을 다른 폴더에 저장하고 싶을 수 있으므로, '워크스페이스' 폴더를 추가적으로 맵핑할 수 있음.

- YAML 에디터에 `- "/workspace:/opt/workspace"`와 같이 추가하여 컨테이너 내부의 `/opt/workspace`와 컨테이너 외부의 `/workspace`를 싱크하도록 설정함.

- 이 맵핑을 하지 않고 컨테이너 내부에서 `/opt/data` 외의 다른 경로에 폴더를 만들고 작업하면, 컨테이너가 사라질 때 데이터도 사라지게 됨.

- 설정 후 'Deploy'를 다시 눌러 변경사항을 적용함.

- 
        <strong>VS Code 연동 (SSH 키 활용):</strong>

- 터미널에서 파일을 확인하는 것이 불편할 경우, VS Code와 연동하여 생성되는 파일을 실시간으로 확인할 수 있는 구조를 만듦.

- <strong>루트 엑세스 (SSH) 확인:</strong> VPS 오버뷰 화면에서 '루트 엑세스' 정보를 복사함. 로컬 VS Code 터미널에서 `ssh root@<IP 주소>` 명령어를 실행하면 접속할 수 있으나, 처음에는 패스워드를 요청하고 보안에 취약할 수 있음.

- <strong>SSH 키 설정 (권장):</strong>

- 호스팅어 설정 페이지에서 'SSH 키'를 선택하고 '+ SSH 키'를 누름.

- 로컬 터미널에서 `ssh-keygen -t rsa -b 4096 -C "유니크한_키_값"` 명령어를 실행하여 SSH 키를 생성함. 메일 주소 대신 본인이 인지하기 좋은 고유 키 값을 넣음.

- 파일 패스를 선택하라고 뜨면 Enter를 눌러 기본 경로를 사용하고, '패스프레이즈' 설정을 요구하면 설정하여 보안을 강화함.

- `cat ~/.ssh/id_rsa.pub` 명령어를 사용하여 생성된 퍼블릭 키의 내용을 확인하고 복사함.

- 복사한 키 내용을 호스팅어 SSH 키 설정 화면의 'SSH 키 컨텐트'에 붙여넣고 저장함. SSH 키 설정이 완료됨.

- <strong>SSH 환경 설정 수정:</strong>

- 로컬 터미널에서 `nano ~/.ssh/config` 명령어로 SSH 설정 파일을 염.

- 아래와 같이 설정을 추가함:
                        <pre>
Host 닉네임 (예: hermis-vps)
  HostName <VPS의_IP_주소>
  User root
  IdentityFile ~/.ssh/id_rsa
  Port 22
                        </pre>

- `Ctrl+O`를 눌러 저장하고 (`파일_네임투라이트`가 뜨면 Enter), `Ctrl+X`를 눌러 나노 에디터를 종료함.

- <strong>VS Code에서 원격 접속:</strong>

- VS Code의 확장(Extension) 아이콘에서 'Remote SSH'를 검색하여 설치함.

- `Cmd+Shift+P` (또는 `Ctrl+Shift+P`)를 눌러 커맨드 팔레트를 열고 'Remote SSH: Connect to Host...'를 선택함.

- 설정했던 닉네임(예: `hermis-vps`)을 선택함.

- 처음 접속 시 패스프레이즈를 입력하라고 뜨면 입력함.

- VS Code가 VPS 서버로 접속됨. 'Open Folder'를 통해 `/docker/hermes-agent-.../data` (또는 맵핑한 `/workspace`) 폴더를 열어 Hermis Agent의 파일들을 실시간으로 확인할 수 있음.

- 이를 통해 텔레그램 등으로 요청한 작업 결과물, 스킬 생성 여부 등을 확인하고 피드백을 줄 수 있음.

- <strong>보안 강화 (선택 사항):</strong>

- Tailscale 같은 서비스를 설치하여 퍼블릭 SSH 접속을 차단하고 VPN으로만 접속하게 구현할 수도 있음.

- 이러한 설정 방법은 Hermis Agent에게 직접 질문하여 도움을 받을 수 있음 (예: "Tailscale로 VPN으로만 접속하게 하고 싶은데, 호스팅어 VPS 연결 상태에서 어떻게 설정해야 해?").

## **Hermis Agent 활용 방법 및 자동화**

- 
        <strong>Hermis Agent의 기본 파일 구조 및 역할:</strong>

- 도커 컨테이너로 설치한 경우, `/opt/data` (컨테이너 내부)의 내용이 `/data` (VPS 호스트 볼륨)에 저장됨.

- <strong>`config` 파일:</strong> 설정 파일.

- <strong>`api_keys` 파일:</strong> API 키를 넣어주는 파일.

- <strong>`soul.md`:</strong> 비서의 이름, 성격, 역할, 목표, 말투 등을 부여하는 핵심 파일임 (클로드 코드/코덱스의 역할 프롬프트와 유사).

- <strong>`memories` 폴더:</strong>

- <strong>`user.md`:</strong> 사용자(나)에 대한 정보를 정리하는 문서로, Hermis Agent가 사용자와의 대화 중 자동으로 생성하고 기록함.

- <strong>`memory.md`:</strong> Hermis Agent 자체의 기억력으로, 기억해야 할 내용들을 적어두면 장기 기억으로 활용함.

- 
        <strong>`soul.md` 설정 및 테스트:</strong>

- VS Code로 접속하여 `soul.md` 파일을 수정함 (예: 이름은 자비스, 24시간 업무 관리 AI 비서, 목표, 역할 등을 설정).

- 수정 후, 텔레그램 등에서 `/new` 명령어를 입력하여 세션을 다시 로드함.

- 봇에게 "이름이 뭐야?"라고 질문하면, 수정된 `soul.md` 내용을 기반으로 "제 이름은 자비스입니다. 부시 대표님의 24시간 업무 관리 AI 비서로 도와드리겠습니다."와 같이 응답하는 것을 확인 가능함.

- 
        <strong>사용자 정보 (`user.md`) 자동 생성 및 기록:</strong>

- Hermis Agent를 활용할 때 가장 먼저 하면 좋은 작업은 '내 비서로서 일을 잘하기 위해 나에 대해서 궁금한 사항을 질문해 줘'라고 요청하는 것임.

- AI 비서는 개인 비서와 유사하므로, 처음에는 비서가 사용자에 대해 잘 알아야 업무를 잘 해 줄 수 있기 때문임.

- AI 비서가 질문을 하면, 사용자는 이에 대해 친절하게 답변함 (최대한 많은 정보를 제공하는 것을 추천함, 실제 직원을 채용한다고 생각하고 1~2시간 할애하여 미팅하듯이 알려주는 것이 좋음).

- 사용자가 답변하면 Hermis Agent는 자동으로 `memories/user.md` 파일에 해당 정보를 생성하고 기록하기 시작함.

- `memories/user.md` 파일이 생성되어 사용자에 대한 정보가 자동으로 추가되고, AI 비서는 이 정보를 항상 기억하면서 작업을 수행함.

- 
        <strong>Hermis Agent 활용 프로세스:</strong>

- 1. `soul.md`를 활용하여 비서의 역할, 업무 범위, 말투 등을 설정함.

- 2. 데이터 입력: AI 비서가 사용자 정보를 최대한 능동적으로 질문하게 만들고, 사용자가 친절하게 답변하여 `user.md`에 기록함.

- 3. 추가로, 어떤 업무를 어떤 식으로 할 것인지 담당할 업무에 대해 논의함 (예: 간단한 업무부터 시작).

- 4. 업무 프로세스가 어느 정도 정형화되면, AI 비서가 이를 '스킬'로 자동 생성함.

- 5. 자동 생성된 스킬을 활용하고, 필요한 경우 자동화 작업까지 설정함 (예: 특정 주기로 작업 수행 및 보고).

- 
        <strong>리서치 작업 요청 및 스킬 자동 생성 예시:</strong>

- 사용자가 '리서치 작업을 해주는 프로세스를 셋업하고 싶다. 핵심 결론 세 줄, 주의사항 포함'과 같이 요청함.

- Hermis Agent는 해당 요청을 `memories/user.md`에 추가하고, 리서치 보고 프로세스를 생성함.

- 주제를 던져줌 (예: '소규모 팀에서 AI 비서를 도입할 때 가장 먼저 자동화하면 좋은 업무').

- Hermis Agent는 작업 내용, 툴 사용 과정 등을 실시간으로 터미널에 출력하며 리서치를 진행함.

- 리서치 완료 후, 요청한 구조에 맞춰 내용을 정리하여 보고서를 생성하고 출처까지 포함함.

- <strong>스킬 자동 생성:</strong> Hermis Agent는 'Self-Improvement Review'를 통해 'Single Topic Research Brief'라는 스킬을 자동으로 생성함. 이는 AI 직원이 스스로 작업 매뉴얼을 생성하는 것과 같음.

- 
        <strong>워크스페이스 폴더에 PDF 보고서 생성 요청 및 피드백:</strong>

- 사용자가 '리서치 작업할 때는 워크스페이스 폴더 안에 PDF 파일로도 보고서를 제작해서 생성해 주면 좋겠다'고 요청함.

- 처음에는 권한 문제로 워크스페이스 외부 경로에 파일을 생성하지 못할 수 있음.

- 이 경우 AI 비서에게 권한 부여를 직접 지시함 (예: "쓰기 권한이 없다고 하는데 권한 부여를 직접 시켜줄게").

- AI 비서가 권한 요청을 하면 'Allow Once' 등으로 권한을 부여함.

- 권한 부여 후 다시 PDF 파일 생성을 테스트해 달라고 요청하면, `/opt/workspace` (맵핑된 외부 `/workspace` 폴더) 안에 PDF 파일을 생성하는 것을 확인할 수 있음.

- 생성된 PDF의 디자인 등은 추가 피드백을 통해 개선할 수 있음.

- 
        <strong>자동화 세팅 (주기적 리서치 요청):</strong>

- 리서치 작업을 요청할 때만 하는 것이 아니라, 'AI 에이전트 관련해서는 매주 월요일 오전 10시(한국 시간 기준)에 리서치해서 알려줬으면 좋겠다'고 요청함.

- Hermis Agent는 이 요청만으로 자동화 세팅(크론 설정)을 진행함.

- 크론 설정이 완료되면, UTC 기준이 아닌 한국 시간 기준으로 오전 10시에 설정되도록 요청하면 AI 비서가 이를 반영하여 다시 설정함.

- 
        <strong>Hermis Agent의 차별성 요약:</strong>

- 클로드 코드나 코덱스는 사용자가 직접 앉아서 쓰는 강력한 에이전트에 가깝다면, Hermis Agent는 한 단계 더 나아가 사용자의 선호도나 업무 형식을 기억하는 '비서'처럼 활용할 수 있음.

- 텔레그램 등에서 AI 비서와 인터랙션 하는 과정을 통해, 정말 비서에게 요청하고 비서가 작업을 대신해 주는 느낌을 받을 수 있음.

- 
        <strong>확장 가능성:</strong>

- 단순히 비서 한 명을 넘어, 리서치 담당, 콘텐츠 담당, 마케팅 담당, 법무 담당 등 여러 직무별 AI 팀원을 구성하는 '작은 조직'을 설계하는 단계로 확장하여 활용할 수 있음.

- 오늘 영상을 통해 먼저 나만의 개인 비서를 만들어 보고, 유용하다면 조직 관점에서 직무별로 에이전트를 추가하여 '멀티 에이전트' 형식으로 활용해 보는 것도 좋음.

- 멀티 에이전트 방식에 대한 관심이 있다면 후속 영상으로 다룰 예정임.
