## **Vibe Editing 및 Remotion 소개**

- Vibe 코딩, Vibe 자동화에 이어 Vibe 편집이 등장함.

- 이는 온라인 비즈니스 구축 및 수익 창출을 원하는 사람들에게 매우 좋은 소식으로 여겨짐.

- 이 영상에서 Simon이라는 친구가 Remotion이 무엇인지, 어떻게 작동하는지, 몇 분 안에 Claude Code를 사용하여 자신만의 비디오를 편집하는 방법을 보여주기 위해 초대됨.

- Simon은 Remotion과 Claude Code의 조합을 통해 마치 숙련된 편집자가 팀에 있는 것처럼 비디오를 생성할 수 있는 최첨단 기술을 선보임.

- 최근 데모를 통해 HeyGen과 Remotion을 결합하여 AI 아바타가 등장하고 배경에 모션 그래픽이 있는 비디오를 프롬프트만으로 만들 수 있음을 보여줌.

- 스크린 앞에 있을 필요조차 없어짐.

- 흥미로운 설명자도 옆에 배치 가능함.

- 사람들은 또한 '나노 바나나' 또는 '클링' 같은 것을 활용하여 영상을 만들고, Remotion을 사용하여 이를 하나로 엮어 영화 같은 장면과 영상을 제작함.

- 가속기(accelerator)를 위해 만든 예시에서는 하나의 프롬프트로 이름, 로고 등을 찾아 재미있는 모션 설명 비디오를 만듦.

## **Remotion이란 무엇인가**

- Remotion은 비디오 편집 또는 모든 구성 요소나 모션 그래픽 생성을 코드로 변환하는 서비스임.

- 특히 Claude 4.5 Opus 모델이 코드 생성에 매우 뛰어나기 때문에, 이 Remotion 서비스를 활용하여 프롬프트만으로 원하는 비디오를 생성할 수 있게 됨.

## **Remotion의 작동 방식: 자산 처리 및 레이어링**

- 비디오 편집기들은 백그라운드에서 코드를 사용함.

- Remotion은 Claude 프로젝트를 설정할 때 A-roll, 기타 자산(예: Liam의 얼굴, 이미지)을 덤프해두면 Claude가 이러한 자산을 인식하고 필요한 곳에 임베드함.

- 예를 들어, HeyGen 비디오에서 자신을 오른쪽에 배치하고 싶다면 Remotion이 포지셔닝 및 레이어링 기능을 제공하여 원하는 위치에 자신을 배치할 수 있게 함.

- 본질적으로 Remotion은 코드 기반 비디오 편집기로 작동함.

## **Claude에 Remotion 스킬 가르치기**

- Remotion을 효과적으로 사용하기 위한 핵심 이론은 Claude에게 '스킬'을 가르치는 것에 있음.

- 이는 포켓몬에게 물을 건너거나 나무를 자르기 위해 '파도타기'나 '풀베기' 같은 기술을 가르치는 것과 유사함.

- Claude에게도 "여기 비디오 편집 스킬이 있고, Remotion을 사용하여 이 작업을 수행할 것"이라고 가르침.

- 스킬 학습은 GTA에서 치트 코드를 입력하는 것처럼 매우 쉬운 과정임.

- 특정 명령어를 Claude에 입력하면 Claude가 해당 스킬을 자동으로 학습함.

## **Remotion 프로젝트 설정 및 비디오 생성 시연**

- Claude Desktop 앱을 사용함.

- 새 폴더를 만들고 'remotion Liam' 등으로 이름을 지정함.

- Claude 앱에서 '코드' 버튼을 눌러 코드 모드로 전환함.

- 제공된 스킬 명령어를 붙여넣어 Remotion 스킬을 설치함.

- Claude에게 다음과 같은 프롬프트를 입력함: "Liam의 AAA accelerator를 위한 비디오를 만들려고 함. 웹에서 Liam의 AAA accelerator를 검색하여 색상, 브랜딩 등 모든 정보를 가져와 이것이 무엇인지 설명하는 비디오를 만들어줘. 단, 먼저 위에 제공된 스킬을 설치하고 Remotion으로 프로젝트를 생성하여 비디오를 제공해줘."

- 이 프롬프트는 의도적으로 모호하게 작성되었으며, `ask`, `code`, `plan` 옵션 중 `code`를 선택하여 실행함.

- 처음에는 설치가 실패했으나, Claude는 계속해서 새로운 시도를 함.

## **AI 모델의 코드 투자 이유와 미래의 소프트웨어**

- Claude, GPT, Gemini와 같은 AI 모델이 코드에 막대한 투자를 하는 이유는, 코딩을 통해 컴퓨터에서 기술적으로 모든 것을 할 수 있기 때문임.

- 컴퓨터의 모든 것은 코드이며, 이를 충분히 잘 다루면 인간이 키보드와 눈을 사용하는 것과 같은 자율적인 작업을 컴퓨터가 수행할 수 있음.

- SAS(Software as a Service)의 미래는 인터페이스 접근 여부에 따라 달라질 수 있음.

- 미래에는 사용자가 직접 SAS 플랫폼에 로그인하여 기능을 사용하는 대신, Claude Code 또는 다른 에이전트 인터페이스를 통해 SAS의 핵심 기능에 접근하는 방식이 될 것으로 보임.

- A16Z(Andreessen Horowitz)는 웹의 미래를 '에이전트 중심(agentic)'으로 보고 투자를 하고 있음.

- UI를 구축하기보다 다른 에이전트가 사용할 수 있는 웹사이트와 앱을 만드는 것에 중점을 둠.

- 예: 남아프리카 이벤트 항공권 구매 시, 직접 Google Flights에 가는 대신 AI 에이전트에게 가장 저렴한 티켓을 찾아달라고 요청하여 구매하는 방식.

- 결국 사용자는 결과물을 얻는 것에만 관심이 있기 때문임.

## **Remotion의 UI 및 미세 조정, 자동화된 워크플로우**

- Remotion 자체에는 로컬에서 실행되는 비디오 편집기가 있어 세부 조정을 할 수 있음.

- 이 편집기에는 비디오를 렌더링하고 자산을 관리하는 공간이 있음.

- Simon의 로고, Liam의 얼굴 등은 Remotion이 자동으로 스크래핑하여 사용함.

- Remotion이 이 모든 것을 조합하여 보이스오버와 함께 비디오를 생성함.

- 필요하다면 직접 들어가서 세부 사항을 조정할 수 있으나, 프롬프트를 통해서도 조정 가능함.

- 'Instantly' 같은 콜드 이메일 캠페인 설정 예시처럼, AI 에이전트가 마케팅 데이터 분석, 새로운 전략 수립, 캠페인 계획, 이메일 카피 작성, AB 테스트 계획까지 모두 처리하고, 'Instantly' 같은 툴을 통해 캠페인을 설정하도록 지시할 수 있음.

- SAS 플랫폼은 이제 확장된 콜드 이메일 캠페인 푸시, 이메일 발송 도메인 및 계정 제공과 같은 핵심 기능을 제공하는 역할만 하게 됨.

- Booked.in 플랫폼 구축 과정에서도 MCP(음성 에이전트)를 활용하여 "이 회사를 위한 음성 에이전트를 만들어 매일 전화를 걸어줘"와 같은 명령으로 자동화를 실험했음.

- 모든 작은 작업들이 자동화되고 있지만, 어떤 소프트웨어가 최고의 결과물을 내는지 아는 것은 여전히 사용자에게 달려있음.

## **생성된 비디오 예시 및 수정 시연**

- 첫 번째 시도에서 생성된 비디오는 "AAA accelerator"에 대한 정보를 요청했음에도 불구하고, Liam의 "무료 커뮤니티"(280K 회원) 정보와 혼동하여 내용을 만듦.

- 비디오 편집 요청 시 "AAA accelerator" 텍스트를 "AI automation agency hub"로 변경해달라고 프롬프트에 입력함.

- "Liam에게는 두 개의 커뮤니티가 있으니, AAA accelerator가 아닌 Liam의 AI automation agency hub가 언급되도록 해줘"와 같이 더 구체적인 프롬프트를 제공함.

- 스크린샷을 찍어 특정 장면의 텍스트를 "scale to $1 billion a month"로 변경해달라고 요청할 수도 있음.

- Claude는 이러한 요청을 받아들여 수정 작업을 진행함.

## **Remotion의 활용 사례: 개인 생산성 및 서비스 제공**

- <strong>개인 생산성</strong>:

- 개인적으로 비디오를 편집하고 모션 그래픽을 생성하는 데 Remotion을 활용할 수 있음.

- Simon은 자신의 리드 생성 비디오 전체를 Remotion으로 편집했다고 언급함.

- HeyGen과 같은 AI 아바타도 사용하여 영상 제작이 가능함.

- <strong>서비스 제공 (AI 에이전시)</strong>:

- SAS 기업에게 제품 데모 비디오를 제공할 수 있음.

- 마케팅 에이전시에게 제품 관련 비디오를 제작해 줄 수 있음.

- 이커머스 에이전시에게 Remotion을 사용하여 제품 자산을 삽입하고, "멋진 광고를 만들었으니, 앞으로 30일 동안 매일 광고를 만들어주고 X달러를 받을 수 있을까?"와 같이 제안하여 서비스를 제공할 수 있음.

- <strong>워크스페이스를 통한 서비스 모델</strong>:

- 이커머스 클라이언트나 에이전시 클라이언트를 위해 워크스페이스(폴더)를 설정함.

- 폴더 안에 제품 사진, 제품 이름, 브랜드 가이드라인(색상, 웹사이트 브랜딩, 타이포그래피, 헥스 코드 등) 정보를 담아둠.

- Claude Code 내에서 새로운 광고 크리에이티브를 브레인스토밍하고, AI가 이 모든 정보를 고려하여 특정 제품을 참조하고, 계획을 바탕으로 새로운 비디오 광고를 생성하게 함.

- Remotion은 창의성과 워크플로우를 보조하는 도구이며, 그래픽 변경이나 비디오 편집이 필요한 모든 작업에 활용될 수 있음.

- 이는 모션 그래픽을 훨씬 쉽게 구축할 수 있도록 함.

## **모션 그래픽의 가능성 및 스킬 스태킹**

- Remotion으로 제품 데모 비디오 제작이 가능함.

- Booked.in의 제품 데모 예시에서는 UI 자산을 같은 폴더에 넣어 Remotion이 UI 구성 요소를 고려한 영상을 제작함.

- 이는 회전(swivels), 지도 애니메이션, 3D 효과 등 다양한 모션 그래픽을 포함할 수 있음.

- AI는 크리에이티브가 아닌, 도구로서의 역할을 함. Rick Rubin이 악기를 연주할 줄 모르지만 '취향'을 아는 것처럼, AI 사용자는 '취향'을 갖는 것이 중요함.

- <strong>스킬 스태킹(Skill Stacking)</strong>:

- 단순한 제품 데모를 넘어 여러 스킬을 결합하여 복잡한 영상을 만들 수 있음.

- 예를 들어, Sora 2 또는 V3로 사실적인 비디오 클립을 다량 생성한 후, Remotion을 사용하여 이 클립들을 스티칭하고 음악을 추가하여 하나의 영상으로 만듦.

- <strong>복합적인 워크플로우 예시</strong>:

- '인터스텔라 테마'의 제품 데모를 만들고 싶을 경우, 먼저 nanobanana 같은 도구로 테마에 맞는 이미지를 생성함.

- 생성된 이미지를 V3 또는 Cling과 같은 AI 이미지-비디오 모델에 넣어 비디오로 변환함.

- 이 비디오들을 Remotion에 넣어 스티칭하고, 필요한 경우 Remotion으로 생성된 레이어(예: 작은 그래픽 요소)를 비디오 위에 겹쳐서 최종 영상을 만듦.

- Remotion은 A-roll과 모든 자산을 활용할 수 있는 프롬프트 기반 비디오 편집기임.

- Claude Code와 자산을 연동하여 브랜드 가이드라인을 따르는 일관된 테마의 영상을 만들 수 있음.

- 마케팅 브리프가 있다면, 이를 `claude.md` 파일 안에 넣어 Claude가 항상 그 스타일을 일관되게 유지하도록 지시할 수 있음.

## **시작 가이드 및 미래 전망**

- <strong>Remotion 시작 가이드</strong>:

- <strong>초보자</strong>: Claude Desktop 애플리케이션을 사용하는 것이 가장 좋음. 이는 Claude와 동일한 인터페이스를 제공하여 모든 것을 쉽게 만듦.

- <strong>숙련자</strong>: VS Code와 Claude Code를 함께 사용하면 측면에서 모든 자산을 볼 수 있어 관리가 더 쉬움.

- Claude Code 생태계는 폭발적으로 성장하고 있으며, 12개월 후의 변화를 예측하기 어려울 정도로 빠르게 발전 중임.

- Mac Mini와 같은 기기를 활용하여 즉석에서 비디오 및 마케팅 자산을 만들 수 있는 시대가 옴.

- 이 기술을 가지고 실험하고 즐기면, 필요한 사람들에게 이를 판매할 수 있을 것임.
