## **AI 직원 Mis Agent 활용의 새로운 가능성: 데스크톱 앱과 외부 서버 연동**

- 기존 Mis Agent는 반복 작업 실행, 보고, 텔레그램/슬랙 메신저를 통한 외부 작업 지시 등 유용하게 활용되어 왔음.

- 다만 Claude Code나 Codex와 같은 데스크톱 앱이 없어 PC에서 실시간 작업 시 활용에 아쉬움이 있었음.

- 최근 Hermes에서 이러한 니즈를 파악하여 데스크톱 앱을 출시함.

- 데스크톱 앱에서 여러 설정 수정 가능함.

- 직원 지정 및 여러 세션 띄워 작업 요청 가능함.

- 생성된 파일 즉시 확인 및 관리 가능함.

- 본 영상에서는 Mis 데스크톱 앱 설치 및 활용 방법을 안내함.

- 특히 Hermes Agent는 24시간 켜두고 직원처럼 활용하는 것이 핵심이므로, 데스크톱 앱을 로컬 PC에서만 돌리는 것이 아닌 외부 서버와 연동하는 방식이 중요함.

- 모든 사용자가 고가인 Mac mini를 구매하기 어렵기 때문에, Hostinger VPS를 활용하여 설치하고 연동하는 방법을 다룸.

- Hostinger의 제작 지원을 받아 VPS 설치 및 연동 방법을 소개함.

- 할인 쿠폰(CITIZENDEVSE)을 제공하여 외부 서버에 AI 직원을 손쉽게 설치하고 데스크톱 앱으로 연동하여 활용하고 싶은 이들에게 시스템 구축을 권장함.

## **구축 시스템의 세 가지 핵심 요소**

- 우리가 구축할 시스템은 세 가지 요소로 구성됨.
        <ol>

- <strong>내 노트북에 Mis 데스크톱 앱 설치:</strong>

- 조종석과 같은 역할 수행함.

- 실제 Mis를 실행하는 곳이 아니라, 화면 및 조작 세팅을 담당함.

- <strong>Mis 백엔드 (엔진 역할) 원격 서버(Hostinger VPS)에 설치:</strong>

- 실제 작업은 Mis 백엔드에서 진행됨.

- Hostinger VPS에 Hermes를 설치하여 여기서 실제 작업을 구동함.

- <strong>Tailscale을 통한 사설망 연결:</strong>

- 내 노트북과 VPS를 Tailscale이라는 사설망으로 연결함.

- 사용자는 노트북의 Mis 데스크톱 앱에서 조종하고, 실제 작업은 VPS 컴퓨터에서 돌아가는 구조임.

        </ol>

- <strong>외부 서버 설치의 필요성:</strong>

- Mis 데스크톱 앱을 로컬에 설치하여 바로 사용할 수도 있음.

- 하지만 로컬 사용 시 몇 가지 단점이 존재함.

- PC를 끄거나 재부팅 시 작동 중지 및 장기 작업에 에러 발생함.

- 무거운 작업 실행 시 PC 성능에 따라 느려질 수 있음.

- 기기 변경 시 다시 설정 이동하는 번거로움 발생함.

- 보안 관점에서, 모든 개인 자료가 노트북에 있는 상태에서 Hermes Agent가 모든 정보에 접근할 수 있어 리스크 발생함.

- 대부분의 경우, Mis Agent는 24시간 켜진 외부 머신에 설치하는 것을 권장함.

- 노트북이 꺼져도 계속 작업 가능함.

- 무거운 작업을 서버에서 전담 가능함.

- 보안 측면에서 필요한 정보만 외부에 제공하여 효율적임.

- <strong>외부 서버 선택 방식:</strong>

- <strong>Mac mini:</strong>

- 가성비 좋은 외부 서버 기기로 많이 활용됨.

- 환율 및 메모리 수요 증가로 가격이 상승하고 있어 Hermes Agent 하나 때문에 구매하기 어려울 수 있음.

- <strong>VPS (Virtual Private Server):</strong>

- 하드웨어 구매 없이 월 구독제로 이용 가능하여 합리적임.

- 특히 Hostinger VPS는 초보자도 Mis를 쉽게 배포하고 활용할 수 있음 (클릭 몇 번으로 설치 가능함).

## **Hostinger VPS를 통한 Mis Agent 설치 및 설정**

- <strong>Hostinger VPS 구매 및 초기 설정:</strong>

- `hostinger.com/citizendevs`로 접속함.

- 두 가지 구독 상품 중 <strong>VPS Hermes Agent</strong> 사용을 추천함 (관리형보다 추후 커스텀 및 다른 서비스 연동에 용이함).

- 사양은 <strong>KVM2</strong>로도 충분함 (로컬 모델 구동이 아닌 API 연결 방식 활용 시).

- 플랜 선택 시, 12개월 또는 24개월 플랜이 할인 폭이 커서 장기 사용 시 유리함.

- 추천 기능 (AI 등)은 별도 비용이므로 필요 없으면 해지함.

- 서버 위치는 말레이시아, 인도네시아, 인도 중 지연 시간이 짧은 곳으로 선택함.

- OS는 <strong>Ubuntu 24.04</strong>를 선택함 (일반 OS).

- Docker 애플리케이션은 나중에 직접 설치하는 것을 보여주기 위해 미리 설치하지 않음.

- 쿠폰 코드 <strong>CITIZENDEVSE</strong>를 입력하여 10% 추가 할인 적용함.

- 기본 세팅으로 루트 비밀번호 자동 생성 요청하고, 멀웨어 스캐너 및 Docker 관리자 추가하여 설정 완료함.

- <strong>Mis Agent 원클릭 배포:</strong>

- VPS 관리 버튼을 눌러 콘솔로 진입함.

- 왼쪽 메뉴에서 <strong>도커 매니저 (Docker Manager)</strong> 선택함.

- 컴포즈(Compose) 섹션의 화살표를 눌러 <strong>원클릭 배포 (One-click deployment)</strong> 선택함.

- 검색 템플릿에서 'mis'를 검색하여 <strong>Mis Agent</strong>를 선택함.

- 어드민 유저네임(예: mis)과 패스워드를 설정하고 별도로 저장함.

- <strong>배포 (Deploy)</strong> 버튼을 눌러 프로젝트로 Mis를 설치함 (Docker 컨테이너에 설치되는 방식임).

- 실행 중으로 뜨면 설치 완료된 것이며, 관리 버튼을 눌러 세부 사항 확인 가능함.

- <strong>Mis Agent 설치 확인 및 초기 설정 (API 연동):</strong>

- 세부 정보에서 4860 포트 링크를 클릭하여 Mis Agent 로그인 화면으로 접속함.

- 설정한 어드민 아이디와 패스워드를 입력하여 대시보드에 로그인함.

- 현재 모델 설정이 되어 있지 않으므로 <strong>OpenAI 코덱스(ChatGPT)</strong> API를 연동함.

- 키(Keys) 섹션에서 OpenAI의 OAuth 설정이 필요함.

- 대시보드에서 직접 설정 불가하며, 도커 컨테이너 터미널에서 커맨드를 실행해야 함.

- <strong>터미널 접속 및 OpenAI OAuth 설정:</strong>

- 도커 매니저의 프로젝트 관리 페이지로 돌아와 터미널 섹션을 클릭함 (컨테이너 내부 터미널).

- 제공된 커맨드를 터미널에 입력하여 실행함.

- 생성된 URL로 접속하여 본인의 ChatGPT 계정으로 로그인하고 코드를 입력하여 연동함.

- <strong>모델 변경:</strong>

- 대시보드로 돌아와 새로고침하면 OpenAI가 연동된 것을 확인 가능함.

- 모델(Models) 섹션에서 <strong>OpenAI Codex 5.5</strong>로 스위치하여 로드함.

- GPT 5.5로 설정된 것을 확인 가능함.

- <strong>Mis Agent 대시보드 주요 기능:</strong>

- <strong>파일:</strong> `opt/data` 폴더 내 파일들을 확인 가능하며, 폴더 생성 및 파일 관리가 가능함.

- <strong>스킬/플러그인:</strong> 에이전트가 활용할 수 있는 스킬 및 플러그인 확인 및 설정 가능함.

- <strong>채널:</strong> 여러 툴(텔레그램, 슬랙 등) 연결 가능함.

- <strong>프로필 (Profiles):</strong>

- 디폴트 프로필의 디스크립션 및 Soul MD (에이전트 성격 및 작동 방식 지정) 설정 가능함.

- 새로운 프로필 생성 또는 스킬/MCP 연동하여 프로필 빌드 가능함.

- <strong>챗 (Chat):</strong>

- 새 챗을 시작하고 프로필 설정에 따라 답변이 달라지는 것을 확인 가능함 (예: "자비스"로 설정 시 "저는 자비스입니다"라고 답변).

## **Mis 데스크톱 앱 설치 및 원격 서버 연동**

- <strong>Mis 데스크톱 앱 설치:</strong>

- `ai.net/desktop`에 접속하여 운영체제에 맞는 설치 파일을 다운로드하고 설치함.

- 설치 시 로컬 Mis도 함께 설치되어 바로 활용 가능함.

- <strong>데스크톱 앱 인터페이스 설명:</strong>

- Claude Code나 Codex와 유사한 인터페이스 제공함.

- 왼쪽 패널: 세션 생성, 대화, 모델 선택, 스킬 확인, 메신저 툴 연동, 아티팩트 확인, 직원(프로필) 선택/생성 가능함.

- 오른쪽 패널: 파일 확인 (나중에 연동 시), 터미널 버튼 있음.

- 우측 상단 <strong>설정 (Settings)</strong> 아이콘: 모델, 채팅, 테마, 워크스페이스, 세이프티, 메모리, 보이스 등 세부 설정 가능함.

- <strong>프로바이더 (Provider)</strong> 섹션: 모델 설정, API 키 (풀 관련) 관리, MCP 문서 추가 가능함.

- <strong>게이트웨이 (Gateway)</strong> 섹션: 기본적으로 로컬 게이트웨이로 설정되어 로컬 PC의 Hermes Agent를 사용함 (예: "저는 Mis Agent예요"라고 답변).

- <strong>Tailscale을 이용한 VPS와 노트북 연동 (사설망 구축):</strong>

- 24시간 구동을 위해 VPS와 데스크톱 앱을 연동해야 함.

- <strong>Tailscale 설치:</strong>

- `tailscale.com`에 접속하여 본인 OS에 맞는 Tailscale을 다운로드 및 설치함.

- 계정 생성 후 로그인하면 콘솔에 본인의 노트북이 자동으로 'Connected'로 표시됨.

- <strong>Hostinger VPS에 Tailscale 설치:</strong>

- Hostinger VPS 관리 페이지의 우측 상단 <strong>터미널</strong>로 접속함 (루트 터미널).

- 제공된 커맨드를 입력하여 Tailscale을 설치함.

- `tailscale up` 명령어를 실행하면 로그인하라는 링크가 뜨며, 본인 Tailscale 계정으로 로그인하여 VPS를 연동함.

- Tailscale 웹사이트 콘솔에서 VPS (예: SRV 17613217)가 연결된 것을 확인 가능함.

- <strong>Docker 컨테이너 포트 포워딩 설정:</strong>

- 사설망 연결 후 Mis Agent 컨테이너에 접근하려면 포트를 열어줘야 함.

- Hostinger VPS의 도커 매니저 > 설치된 컨테이너 관리 탭 > <strong>야멜 편집기 (YAML Editor)</strong>로 이동함.

- `ports` 섹션에 다음과 같이 포트 매핑을 추가함:

- `"- [Tailscale_IP]:9119:4860"`

- 여기서 `[Tailscale_IP]`는 Tailscale 콘솔에서 확인한 호스팅어 VPS의 IP 주소임.

- `9119`는 Mis 데스크톱 앱에서 접속할 포트, `4860`은 Docker 컨테이너에서 Mis가 사용하는 포트임.

- 상단 <strong>배포 (Deploy)</strong> 버튼을 눌러 변경 사항을 반영함.

- 포트 `9119:4860`이 추가되었는지 확인하여 설정이 제대로 되었는지 확인함.

- <strong>Mis 백엔드 서버 실행 및 데스크톱 앱 연결:</strong>

- 포트 설정 후 Mis Agent 컨테이너에서 백엔드 서버를 실행해야 함.

- Mis Agent 대시보드의 CLI에서 "Mis 데스크톱 앱의 리모트 게이트웨이로 텔스케일 IP에 9119 포트로 연결해서 사용하고 싶다. 백엔드 서버로 열어서 연결하려면 mis serve라는 걸 하면 된다."와 같이 요청함.

- Mis Agent가 해당 요청을 처리하여 `mis serve`를 실행하고 연결할 주소를 알려줌 (기본 포트 9119).

- Mis 데스크톱 앱의 설정 > 게이트웨이 > <strong>리모트 게이트웨이 (Remote Gateway)</strong>를 선택함.

- 알려준 URL (Tailscale IP:9119)을 입력하고 <strong>Save & Reconnect</strong>를 클릭함.

- 로그인 요청이 뜨면 Mis Agent 어드민 아이디와 패스워드를 입력하여 인증함.

- 연동이 완료되면 데스크톱 앱에 기존 VPS에서 진행했던 세션들이 나타남.

## **연동된 Mis 데스크톱 앱의 활용 및 이점**

- <strong>세션 및 프로젝트 관리:</strong>

- 왼쪽 패널에서 세션들을 확인 가능하며, 프로젝트로 전환하여 특정 업무별로 세션을 분류하고 관리 가능함.

- <strong>파일 관리 (VPS 연동):</strong>

- 오른쪽 패널에서 VPS에 저장된 파일들을 바로 확인, 더블 클릭으로 내용 프리뷰, 에디트 버튼으로 수정 가능함.

- <strong>인증 방식 정리:</strong>

- 우리가 진행한 방식은 Tailscale을 활용한 사설망 연결임 (개인 활용 시 권장).

- 내 노트북과 Hostinger VPS를 같은 Tailscale 계정으로 연결하고, IP 활용하여 리모트 게이트웨이 9119 포트로 연동함.

- 어드민 ID와 패스워드를 통한 인증을 사용함.

- 공개 인터넷으로 연동하는 방식도 가능하나, 보안을 위해 다른 서비스의 OAuth 인증 등을 함께 진행하는 것이 안전함. 무인증은 지양함.

- 이전 버전의 Mis Agent 사용 시 `mis serve`가 작동하지 않을 수 있으므로, 최신 버전으로 업데이트 필요함.

- <strong>메시징 툴 연동 (Telegram 예시):</strong>

- 데스크톱 앱의 메시징 버튼에서 직접 텔레그램 연동 설정 가능함.

- <strong>Telegram 봇 생성:</strong> BotFather 계정에서 `/newbot` 명령어로 봇을 생성하고, 봇 토큰을 받음.

- <strong>User ID 확인:</strong> User Info Bot에서 자신의 User ID를 확인함.

- <strong>데스크톱 앱 설정:</strong> 봇 토큰, User ID, `allow_all_users`는 `false`, 홈 채널(내 ID) 등을 입력하고 저장함.

- 게이트웨이 리스타트 요청 후, Mis Agent가 직접 리스타트 처리하여 연동 완료함.

- 텔레그램에서 봇을 시작하면 데스크톱 앱에서 바로 답변을 해주는 것을 확인 가능함.

- <strong>기존 Mis Agent 불러오기 (URL 변경):</strong>

- 새로운 VPS에 Mis Agent를 설치하는 것이 아닌, 기존에 사용하던 Mis Agent (다른 VPS 또는 컨테이너)를 불러올 수도 있음.

- 설정 화면에서 게이트웨이 URL만 해당 Mis Agent의 것으로 수정하고 리커넥트하면 됨.

- 예시로, 이전에 만들었던 '자비스', '헤밍웨이', '뉴턴' 세 명의 직원 프로필을 불러와 선택하여 작업 요청 가능함을 시연함.

- <strong>Mis 데스크톱 앱 활용 시연 (Hemingway 리서치):</strong>

- 헤밍웨이(리서치 담당)에게 작업 요청 시, 왼쪽에 세션이 활성화되고 요청이 들어가며 툴콜, 스킬 확인 등 작업 과정이 하단에 표시됨.

- 슬랙에서 스레드를 여러 개 나누기 어려운 점과 달리, Mis 앱은 왼쪽 세션 플러스를 눌러 여러 작업을 별도로 요청하기 편리함.

- 이전에 슬랙이나 텔레그램으로 외부에서 작업했던 내용도 데스크톱 앱에서 확인하고 이어서 작업 진행 가능함 (전체 Mis 작업 현황 확인 앱).

- 크론 잡(Cron Job) 현황, 실행 시간, 답변 이력 등도 앱에서 바로 확인 가능함.

- 리서치 결과가 테이블 형태로 깔끔하게 정리되어 나타나며, 복사하거나 파일로 생성하여 편집/프리뷰 가능함.

- 실시간으로 함께 작업하는 인터페이스가 편리하며, 스킬 업데이트 및 메모리 연동 등이 싱크되어 모든 플랫폼에서 일관된 활용 가능함.
