## **노션 커스텀 에이전트 소개 및 기존 에이전트와의 차이점**

- 노션 에이전트는 요청 시 노션 내에서 여러 단계를 거쳐 결과물을 만드는 강력한 AI 에이전트임.

- 기존 노션 에이전트의 아쉬운 점은 매번 직접 요청해야 움직인다는 것임.

- 노션에서 메일 확인, 주간 리포트 작성, 프로젝트 관리 문서 업데이트 등 매주 반복되는 업무가 많음.

- 기존에는 이러한 반복 작업을 노션 에이전트에게 요청하려면 매번 직접 지시해야 했음.

- 최근 노션이 커스텀 에이전트 기능을 출시하며 반복 업무 자동화가 가능해졌음.

- 본 영상에서는 노션 커스텀 에이전트를 활용하여 반복 업무를 손쉽게 자동화하는 방법을 소개함.

- 노션의 영상 제작비 지원을 받아 소개하는 기회임을 밝힘.

## **노션 에이전트와 커스텀 에이전트 개념 정리**

- <strong>노션 에이전트:</strong>

- 하단 채팅에서 원하는 것을 요청하면 노션 내 자료 검색, 외부 웹 서치 등을 통해 결과물을 노션에 생성해 주는 AI 에이전트 기능임.

- 꽤 많은 일을 자동화해주지만, 시작점이 항상 사용자 요청이라는 한계가 있음.

- 개인의 노션 권한 안에서 사용하므로 팀 단위보다는 개인 중심으로 활용하는 툴에 가까움.

- <strong>커스텀 에이전트:</strong>

- 좌측 사이드바의 에이전트 섹션에서 사용 가능함.

- 매번 요청하지 않아도 스케줄, 이벤트, 멘션 같은 '트리거'로 알아서 실행됨.

- 반복 업무를 전담하는 직원 같은 역할을 한다고 볼 수 있음.

- 개인 권한이 아닌 에이전트별로 권한 조정을 할 수 있음.

- 팀 내 자산으로 공유 가능하여 개인용보다는 조직 전체가 함께 활용할 수 있는 장점이 있음.

- 가장 큰 차이점은 '트리거'의 유무임.

## **커스텀 에이전트의 다양한 트리거 유형**

- <strong>반복 스케줄링:</strong>

- 매일 특정 시간, 매주 특정 요일 등 원하는 스케줄을 선택하여 반복 작업을 실행할 수 있음.

- <strong>노션 이벤트:</strong>

- 노션에서 특정 이벤트가 발생했을 때 이를 트리거로 워크플로우를 실행할 수 있음.

- 예시: 페이지 생성, 페이지 내 특정 요소(속성) 상태 변경 등.

- <strong>외부 서비스 이벤트:</strong>

- 노션 외에 연결할 수 있는 외부 서비스에서 이벤트가 발생했을 때 커스텀 에이전트를 실행할 수 있음.

- 예시: 슬랙 멘션 발생, 새로운 메일 수신, 캘린더 이벤트 생성 등.

- 본 데모에서는 이 세 가지 트리거를 각각 확인해 볼 예정임.

## **커스텀 에이전트 사용 요금 및 크레딧**

- 커스텀 에이전트는 노션의 고급 기능으로, 비즈니스 및 엔터프라이즈 플랜 전용으로 활용 가능함.

- 기본적으로 월 300 크레딧 정도를 제공함.

- 크레딧제로 운영되며, 추가 사용 시 10달러에 약 1000 크레딧을 구매할 수 있음.

- 기본 무료 크레딧으로도 몇 가지 커스텀 에이전트를 충분히 설정하고 활용해 볼 수 있음.

- 작업이 무거울수록 더 많은 크레딧을 사용하므로, 크레딧 대시보드를 참고하여 효율적으로 에이전트를 구성하고 활용하는 것이 중요함.

## **커스텀 에이전트 생성 방법**

- 노션 왼쪽에 있는 '에이전트' 섹션에서 플러스 버튼을 눌러 커스텀 에이전트를 생성함.

- 두 가지 방식으로 제작 가능함:

- <strong>AI 채팅창 활용:</strong> 채팅창에 원하는 커스텀 에이전트의 기능을 설명하여 제작함.

- <strong>템플릿 활용:</strong> '마켓플레이스 둘러보기'를 통해 이미 생성되어 있는 템플릿 중 원하는 것을 선택하고, 자신의 상황에 맞게 조정하여 활용함.

- 두 가지 방식 모두 데모에서 시연 예정임.

## **데모 1: 템플릿 활용 '아침 브리핑' 에이전트 생성 및 테스트**

- <strong>템플릿 선택:</strong> 마켓플레이스에서 '아침 브리핑' 템플릿을 선택함.

- <strong>구성 및 연동:</strong>

- 템플릿 선택 즉시 에이전트 설명이 나오고, '에이전트 사용하기'를 누르면 자동으로 아침 브리핑 커스텀 에이전트가 구축됨.

- 캘린더와 메일 연결을 요청하며, 플러스 버튼을 눌러 구글 계정 등을 연동함.

- <strong>트리거 설정:</strong>

- 커스텀 에이전트의 자동 실행을 위해 트리거를 설정해야 함.

- 아침 브리핑은 매일 오전 9시와 같이 스케줄 형태로 실행되는 것이 자연스러워 이미 자동으로 설정됨.

- <strong>지침 문서:</strong>

- 지침 문서에 아침 브리핑의 워크플로우, 작업 내용, 참고 지침, 템플릿 등이 작성되어 있음.

- 수동으로 수정하거나 채팅창에 요청하여 지침을 수정할 수 있음.

- <strong>도구와 사용 권한:</strong>

- 가장 중요한 섹션으로, 에이전트가 접근하고 활용할 수 있는 도구와 권한을 제어할 수 있음 (자세한 내용은 추후 설명).

- <strong>테스트 실행:</strong>

- 설정 저장 후 '에이전트 실행하기' 버튼을 눌러 테스트함.

- 왼쪽에 '아침 브리핑 실행을 도와주세요'라는 프롬프트가 자동으로 입력되며 실행됨.

- 결과: '아침 브리핑 허브' 데이터베이스에 새 페이지가 추가되고, 오늘의 최우선 항목 및 어제 놓친 사항 등이 정리된 브리핑이 작성됨.

## **데모 2: AI 채팅으로 직접 생성하는 '사업 문의 리드 관리' 에이전트**

- <strong>생성 목표:</strong> 매일 아침 메일함을 확인하여 강연, 광고, 컨설팅 같은 사업 문의로 보이는 메일만 '세일즈 파이프라인' 데이터베이스에 카드로 추가하고 관련 속성을 채우는 에이전트임.

- <strong>활용 데이터베이스:</strong> '세일즈 파이프라인' 데이터베이스를 활용함 (초기 상태 '검토 전', 이후 '논의 중', '진행 컨펌' 등으로 분류).

- <strong>요청 프롬프트:</strong>

- 새 에이전트 플러스 버튼을 누르고 AI 채팅창에 다음과 같이 요청함: "매일 아침 9시에 받은 편지함에서 안 읽은 메일을 확인하고 강연, 광고, 컨설팅 같은 사업 문의로 보이는 메일만 골라서 세일즈 파이프라인 데이터베이스의 카드로 추가해 줘. 메일의 쓰레드 아이디나 클라이언트 명, 사업 문의 요약, 사업 유형, 상태 등의 속성을 채워 줘."

- <strong>노션의 자동 설정:</strong>

- 요청에 따라 노션 AI가 직접 커스텀 에이전트 설정을 진행하며, '세일즈 파이프라인' 데이터베이스를 불러와 확인하고 셋업함.

- <strong>설정 내용 확인:</strong>

- <strong>트리거:</strong> 요청대로 '매일 아침 9시'로 설정됨.

- <strong>지침:</strong>

- 사업 문의 메일 분류 기준, 처리 순서, 채워야 할 카드 속성(메일 쓰레드 ID, 클라이언트 명, 사업 문의 요약, 사업 유형, 상태 등)에 대한 내용이 구체적으로 정리됨.

- 필요에 따라 추가 수정 요청 가능함.

- <strong>도구와 사용 권한:</strong>

- 기본적으로 안전을 위해 권한 제한이 되어 있음.

- '세일즈 파이프라인' 데이터베이스에는 내용 편집이 허용됨 (수정 요청했기 때문).

- 추가 참고 페이지가 있다면 '추가하기' 버튼으로 지정할 수 있음 (현재는 추가하지 않음).

- 메일은 연결된 구글 계정 메일을 활용하여 체크하도록 설정됨.

- 모델: 다양한 모델 중 선택 가능하며, 비용(크레딧 지출)을 고려할 수 있음. 자동으로 선택하도록 둘 수 있음.

- 웹 접근: 현재는 메일 확인 및 데이터베이스 추가만 필요하므로 비활성화됨. 신뢰할 수 있는 URL을 추가하여 특정 웹 접근만 허용할 수도 있음.

- <strong>테스트 실행:</strong>

- 설정 저장 후 미리 보내 놓은 사업 문의 예시 메일이 제대로 분류되는지 '에이전트 실행하기' 버튼을 눌러 테스트함.

- 결과: '그로우웨어 AI 강연 문의' 스레드를 찾아 '세일즈 파이프라인'에 새 카드로 등록했다고 알림.

- 생성된 페이지 확인: 상태 '검토 전', 사업 유형 '강연', 사업 문의 요약, 메일 쓰레드, 클라이언트 명, 우선순위 등 속성이 잘 정리되어 있었으며, 문의 요약 내용도 구체적으로 정리됨.

- 실제 '세일즈 파이프라인' 데이터베이스에 '검토 전' 상태의 카드가 추가된 것을 확인함.

- <strong>인사이트 확인:</strong>

- 우측 상단 '인사이트 보기'를 통해 실행 이력, 요청 내용, 결과, 크레딧 사용량 등을 확인할 수 있음.

- 자동으로 실행할 때 결과물 퀄리티가 아쉽다면 이력을 확인하여 지침 등을 수정하여 개선할 수 있음.

## **데모 3: 노션 이벤트 트리거 활용 '메일 회신 초안 생성 및 슬랙 알림' 에이전트**

- <strong>생성 목표:</strong> '세일즈 파이프라인' 카드를 확인하고 상태를 '논의 중', '진행 컨펌', '진행 완료', '드롭' 등으로 변경하면, 해당 상태에 맞는 메일 초안을 생성하고 슬랙 채널에 관련 내용을 전송하는 에이전트임.

- <strong>트리거 유형:</strong> 이전 데모와 달리 '세일즈 파이프라인' 데이터베이스의 상태 변경이라는 '노션 이벤트'를 트리거로 사용하므로 별도의 커스텀 에이전트를 생성함.

- <strong>요청 프롬프트:</strong>

- 새 에이전트 플러스 버튼을 누르고 다음과 같이 프롬프트를 입력함: "세일즈 파이프라인 데이터베이스에서 카드 상태가 검토 전에서 다른 것들로 변경이 되면 거기에 맞게 메일 초안 드래프트를 생성해 줘. SOP 문서 데이터베이스를 참고해서 메일을 제작해 줘. 메일 초안을 생성한 다음에는 슬랙 채널 세일즈 채널에 이 프로젝트의 요약 내용을 새 메시지로 보내 줘."

- <strong>노션의 자동 설정:</strong>

- 요청에 따라 노션 AI가 커스텀 에이전트 초안을 바로 제작하며, 슬랙 연동만 수동으로 필요하다고 안내함.

- <strong>설정 내용 확인:</strong>

- <strong>트리거:</strong> 요청대로 '노션 이벤트' 중 '세일즈 파이프라인'의 '속성 업데이트'에서 '상태 속성이 편집'되었을 때 실행되도록 설정됨.

- <strong>지침:</strong>

- 상태 변경 시 초안 드래프트 생성, 슬랙 '세일즈 채널'에 프로젝트 요약 메시지 전송 지시, 'SOP 문서'를 참고하도록 안내됨.

- 가장 중요한 점은 '초안만 생성하고 직접 보내지는 말라'고 명시되어 있어 자동 발송을 방지함.

- <strong>도구와 사용 권한:</strong>

- '세일즈 파이프라인': 실제로 내용을 읽고 쓸 수 있도록 '내용 편집 허용' 권한이 주어짐.

- 'SOP 문서': 수정할 필요 없이 읽기만 하므로 '읽기' 권한이 주어짐.

- 메일: 초안 작성을 위해 메일 권한이 주어져 있으나, '보내기' 권한은 제거하여 메일이 자동으로 발송되지 않도록 설정할 수 있음.

- 슬랙 연결: '연결 추가하기'를 통해 슬랙 워크스페이스에 AI 커넥터를 추가하고, '세일즈 채널'에 읽기 및 쓰기 권한을 제공함.

- <strong>테스트 실행:</strong>

- 슬랙 연동 완료 후, '세일즈 파이프라인' 카드에 접속하여 상태를 '검토 전'에서 '논의 중'으로 변경함.

- 상태 변경 시 트리거가 작동하여 '대기 중인 건'으로 표시되고, 에이전트가 실행됨.

- 강연권이므로 강연 SOP를 로드하고 메일 초안 생성을 진행함. 슬랙 채널에 포스트 작업도 진행함.

- 결과:

- Gmail 임시보관함에 회신 초안 메일이 잘 생성됨.

- 슬랙 '세일즈 채널'에 '세일즈 회신 초안봇'이 해당 프로젝트 건에 대한 요약과 CRM 카드 링크를 포함한 메시지를 전송함.

- 개선점: 현재는 상태가 변경되면 항상 트리거가 되도록 설정되어 있지만, 실제 활용 시에는 메일 생성이 필요한 '논의 중' 또는 '드롭' 같은 특정 상태에서만 트리거되도록 세부 설정하는 것을 권장함.

## **데모 4: 외부 서비스 이벤트 트리거 활용 '슬랙 논의 내용 CRM 카드 업데이트' 에이전트**

- <strong>생성 목표:</strong> 슬랙 '세일즈 채널'에서 특정 키워드(`[노션]`)와 CRM 카드 링크가 포함된 멘션이 있을 때, 해당 쓰레드에서 논의된 내용을 '세일즈 파이프라인' 카드 본문의 '슬랙 논의 메모' 섹션에 자동으로 추가하는 에이전트임.

- <strong>트리거 유형:</strong> 슬랙 멘션이라는 '외부 서비스 이벤트'를 활용하므로 별도 에이전트를 추가함.

- <strong>요청 프롬프트:</strong>

- 새 에이전트 플러스 버튼을 누르고 다음과 같이 프롬프트를 입력함: "세일즈의 쓰레드에서 '[노션]'이라는 대괄호가 포함되면 실행을 해 줘. 그리고 쓰레드 원문에서 포함된 CRM 카드 링크를 확인해 가지고 이 쓰레드에서 논의된 내용을 세일즈 파이프라인 카드 본문에다가 슬랙 논의 메모 섹션으로 추가해 줘."

- <strong>노션의 자동 설정:</strong>

- 요청에 따라 노션 AI가 필요한 페이지들을 확인하고 커스텀 에이전트를 제작함.

- <strong>설정 내용 확인:</strong>

- <strong>트리거:</strong> 슬랙 트리거로 설정되며, 채널은 '세일즈'로 선택됨. 특정 문구 `[노션]`이 포함되었을 때만 작동하도록 설정됨 (키워드 추가 가능).

- <strong>지침:</strong>

- '세일즈 채널'에 `[노션]` 문구가 들어가면 실행하며, 링크를 찾아 '슬랙 논의 메모' 섹션으로 내용을 추가함.

- 새 카드를 만들지 않으며, 쓰레드에 나온 내용만 근거로 작성하도록 함.

- 예외 안전 장치로 CRM 카드 링크가 없을 때는 업데이트하지 말라고 명시됨.

- 노션 AI가 제작하는 지침의 퀄리티가 매우 우수하다고 평가됨.

- <strong>도구와 사용 권한:</strong>

- '세일즈 파이프라인': 접근하여 업데이트할 수 있도록 '내용 편집 허용' 권한이 주어짐.

- 슬랙 '세일즈 채널': 쓰레드를 확인할 수 있도록 '읽기 및 회신' 권한이 주어짐.

- 모델: 자동으로 선택하도록 둠.

- <strong>테스트 실행:</strong>

- 슬랙 '세일즈 채널'로 이동하여 쓰레드에 추가 내용을 남기고 `[노션]` 키워드와 CRM 카드 링크를 포함하여 멘션함.

- 트리거가 작동하여 '논의 메모' 섹션 업데이트가 진행됨.

- 결과: 노션 카드 내 '슬랙 논의 메모' 섹션에 슬랙 쓰레드에서 논의된 내용 (예: 제안 금액 제한, 일정 조정)이 잘 정리되어 추가됨.

- 담당자나 마감일 등 언급 시 별도로 구분하여 적어주지만, 현재 언급이 없으므로 '언급 없음'으로 정리됨.

- <strong>결론:</strong> 커스텀 에이전트를 활용하여 노션과 슬랙을 유기적으로 연결하고, 자동으로 데이터 업데이트 및 효율적인 프로젝트 관리를 할 수 있음.

## **오늘 테스트한 커스텀 에이전트 요약**

- 세 가지 유형의 커스텀 에이전트를 '직원'에 비유하여 제작함:

- <strong>1. 메일 분류 직원:</strong> Gmail에 들어온 메일을 '세일즈 파이프라인'에 자동으로 카드 등록하여 메일 분류를 진행함.

- <strong>2. 회신 초안 작성 및 사업 공유 직원:</strong> 노션에서 카드 상태를 업데이트하기만 하면, 해당 상태에 맞는 회신 초안을 제작하고 '세일즈 슬랙' 채널에 요약 메시지를 전송함.

- <strong>3. 슬랙 논의 메모 등록 직원:</strong> 슬랙에 프로젝트가 등록되고 논의된 내용 중 특정 키워드(`[노션]`)가 포함된 내용을 자동으로 받아 CRM 카드에 메모로 등록함.

- 이러한 시스템 구축을 통해 귀찮은 작업은 커스텀 에이전트에게 위임하고, 핵심 업무에 더 많은 시간을 쏟아 생산성을 극대화할 수 있음.

## **커스텀 에이전트 안전 활용을 위한 체크리스트**

- <strong>1. 최소 권한 원칙:</strong>

- 노션 커스텀 에이전트는 기본적으로 '무권한' 상태로 작동함.

- '도구 액세스' 설정에서 필요한 만큼만 최소한의 권한을 제공하여 AI 모델이 작동하도록 설정할 수 있음.

- 원하는 문서들을 추가하고, 각 문서에 어떤 권한(예: 읽기, 편집)을 줄지 세부적으로 설정 가능함.

- 슬랙 채널 접근 권한 및 웹 접근 시 신뢰할 수 있는 URL도 직접 설정할 수 있음.

- <strong>2. 활동 로그 및 버전 기록 확인:</strong>

- '인사이트 보기'에서 에이전트 실행 이력을 확인하여 어떤 트리거로, 어떤 단계를 거쳐 결과물이 생성되었는지 파악할 수 있음. 이를 통해 설정 개선에 활용 가능함.

- '버전 기록' 기능으로 에이전트 설정 수정 이력이 남고, 이전 버전으로 복원할 수 있어 자동 버전 관리가 제공됨.

- <strong>3. 외부 정보 발신 시 확인 필요 기능:</strong>

- 메일 발송과 같이 외부로 나가는 정보에 대해 '확인 필요' 기능을 활성화하여, 에이전트가 직접 보내기 전에 사용자가 직접 확인하고 승인할 수 있도록 안전 장치를 걸 수 있음.

- <strong>4. 팀 공유 기능:</strong>

- '공유' 버튼을 통해 원하는 인원들과 에이전트를 공유하여 팀원들이 에이전트 탭에서 함께 활용할 수 있음.
