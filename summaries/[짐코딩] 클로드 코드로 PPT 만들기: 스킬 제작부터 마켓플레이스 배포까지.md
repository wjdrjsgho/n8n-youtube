## **Claude Code 스킬을 활용한 웹 PPT 제작 및 공유 과정**

- 
        최근 영상에서 사용한 PPT 제작 방법에 대한 질문을 자주 받음.

- 영상 PPT는 클로드 코드 스킬로 만들었음.

- 이번 영상에서는 클로드 코드 스킬로 PPT를 만드는 방법을 직접 보여줄 예정임.

- 단순 PPT 제작뿐만 아니라, 스킬 제작을 어려워하는 사람들을 위해 '스킬 크리에이터' 스킬을 활용해 공식 문서 기반으로 최적화된 스킬을 직접 만들 예정임.

- 생성된 스킬을 플러그인으로 만들어 여러 프로젝트에서 재사용하거나 다른 팀원들과 공유할 수 있도록 마켓플레이스에 등록하는 과정까지 다룰 예정임.

- 
        이 영상을 통해 얻을 수 있는 정보는 다음과 같음.

- 나만의 PPT 스킬을 만들어 보고 싶은 사람.

- 팀원들과 만든 스킬을 공유하고 싶은 사람.

- 클로드 코드를 개발 외 목적으로 활용해 보고 싶은 사람.

## **엔트로픽 공식 PPTX 스킬의 한계 및 HTML 기반 웹 PPT의 장점**

- 
        엔트로픽 공식 문서에서도 PPT를 만들 수 있는 'PPTX 스킬'을 제공하고 있음.

- 해당 스킬은 PPT 파일을 만들어주는 스킬임.

- 
        하지만 공식 PPTX 스킬의 단점이 있음.

- 퀄리티가 너무 아쉬움.

- 생성 속도가 느림.

- 파일 포맷을 HTML 웹으로 하고 싶어도 PPT 파일로만 생성됨.

- 
        사용하는 방법은 클로드 코드를 사용해 HTML 기반으로 웹 PPT 슬라이드를 만드는 것임.

- HTML 기반이라 퀄리티가 좋음.

- 만드는 속도가 빠름.

- 슬라이드 제작 시간이 크게 단축되었음.

## **스킬 크리에이터 스킬 소개**

- 
        웹 PPT 스킬을 만들 때 사용할 스킬은 '스킬 크리에이터' 스킬임.

- '스킬 크리에이터'는 클로드 코드의 스킬을 만들어주는 스킬임. (스킬이 스킬을 만드는 방식)

- 
        '스킬 크리에이터'의 필요성.

- 스킬을 직접 만들어 본 사람은 공식 문서를 보며 스킬 구조를 맞추는 것이 까다롭다는 것을 알 수 있음.

- 'skill.md' 파일의 적절한 줄 수, 스킬 발동 조건 작성법, 워크플로우 단계 설정 등이 쉽지 않음.

- 공식 문서에 모범 사례가 있지만, 하나하나 찾아서 적용하기 번거로움.

- 
        '스킬 크리에이터'의 장점.

- 공식 문서의 스킬 스펙과 모범 사례를 자동으로 반영해 줌.

- 사용자는 스킬 구조를 일일이 신경 쓸 필요 없이, 원하는 기능에만 집중할 수 있음.

- 스킬에 최적화된 구조는 '스킬 크리에이터'가 만들어 줌.

- 
        기존 스킬 개선에도 유용함.

- 새 스킬 생성뿐만 아니라, 이미 만들어둔 스킬이 아쉬운 부분이 있을 때 '스킬 크리에이터'를 활용해 공식 문서 모범 사례 기반으로 구조를 점검하고 개선해 줌.

- 새로 만들 때도 좋지만, 기존 스킬을 더 좋게 다듬을 때도 매우 유용함.

## **엔트로픽 공식 플러그인 마켓플레이스 탐색**

- 
        엔트로픽은 GitHub에 'anthropic/claude-plugins-official'이라는 공식 플러그인 마켓플레이스 저장소를 운영하고 있음.

- 엔트로픽이 직접 관리하는 공식 플러그인 마켓플레이스임.

- 
        마켓플레이스에는 두 종류의 플러그인이 있음.

- 엔트로픽이 직접 만들고 관리하는 내부 플러그인.

- 파트너사나 커뮤니티가 만든 외부 플러그인.

- 
        내부 플러그인 예시.

- '스킬 크리에이터' 스킬.

- '피처 데브' 스킬.

- '프론트엔드 디자인' 스킬 등 개발에 유용한 플러그인이 많이 있음.

- 
        외부 플러그인 예시.

- '컨텍스트', '깃허브', '슬랙', '슈퍼베이스' 등 다양한 외부 플러그인이 있음.

- 
        공식 마켓플레이스는 클로드 코드를 시작하면 자동으로 등록되어 있어 별도 추가할 필요 없음.

## **마켓플레이스 확인 및 '스킬 크리에이터' 설치**

- 
        클로드 코드에서 `/plugin` 명령어를 입력하면 플러그인 관련 탭들이 나타남.

- 
                각 탭의 기능은 다음과 같음.

- <strong>디스커버(Discover) 탭:</strong> 설치 가능한 플러그인(약 480개)을 탐색하고 설치하는 탭.

- <strong>인스톨드(Installed) 탭:</strong> 이미 설치된 플러그인 목록을 확인하고 관리하는 탭.

- <strong>마켓플레이스(Marketplace) 탭:</strong> 플러그인을 배포하는 마켓플레이스 설정 탭.

- <strong>에러(Error) 탭:</strong> 플러그인 로딩 및 실행 중 발생한 오류를 확인하는 탭.

- 
        '마켓플레이스' 탭으로 이동하여 'Claude Plugins Official' 마켓플레이스를 선택함.

- 'Browse Plugins'를 선택한 후, 스킬 목록에서 '스킬 크리에이터' 스킬을 찾아 설치함.

- 이미 설치되어 있으면 체크 표시가 되어 있음.

- 설치가 안 된 경우, 해당 스킬을 선택하고 원하는 스코프에 설치함.

- 
        설치 완료 후, 세션을 종료하고 다시 실행함.

- `/skill creator`를 입력하면 '스킬 크리에이터' 스킬이 나오는 것을 확인할 수 있음.

## **웹 PPT 스킬 제작을 위한 데이터 수집 (리서치)**

- 
        스킬을 바로 만들기 전에, 문제 해결을 위한 '분석, 수집, 계획, 구현' 순서대로 접근하는 것이 중요함.

- 
        웹 PPT 제작을 위한 데이터 수집 (리서치)을 진행함.

- 순수 HTML, CSS, 자바스크립트만으로 웹 PPT를 만드는 가장 효과적인 방법을 리서치할 예정임.

- 이 리서치 결과가 스킬의 핵심 로직이 됨.

- 
        클로드 코드의 강력한 기능인 서브 에이전트 병렬 활용.

- 여러 에이전트가 동시에 각기 다른 주제를 리서치할 것임.

- 프롬프트 내용은 "순수 HTML, CSS, 자바스크립트만으로 웹 기반 슬라이드 프레젠테이션을 구현하는 최적의 방법을 리서치하라."임.

- PPT 제작 라이브러리인 'Reveal.js'는 참고만 하고, 최종 구현은 순수 바닐라 JS로 가능해야 한다고 명시함.

- 
        프롬프트 입력 및 수정 팁.

- 클로드 코드에서 네 줄 이상 텍스트를 붙여넣기하면 단축해서 표시됨.

- 프롬프트 수정이 필요할 경우, 단축키 'Ctrl+G'를 사용해 프롬프트를 텍스트 편집기에서 열어 수정할 수 있음.

- 
        리서치 프롬프트 상세 내용.

- 현재 스킬을 만드는 것이 아니라 PPT를 만드는 최적의 방법을 리서치하는 것이므로 '스킬 크리에이터'는 제거함.

- 프롬프트는 네 개의 주제를 서브 에이전트로 병렬 리서치하도록 구성되어 있음.

- 이러한 메타 프롬프트 생성 방법은 강의에서 다루고 있음.

- 
        서브 에이전트와 에이전트 팀의 차이점 설명.

- 
                <strong>서브 에이전트:</strong>

- 각자 독립된 컨텍스트에서 작업을 수행함.

- 작업 완료 후 프롬프트를 작성했던 메인 에이전트에게 보고함.

- 서브 에이전트끼리는 서로 대화할 수 없음.

- 메인 에이전트는 각 서브 에이전트의 결과를 받아 종합하고 정리하는 역할을 함.

- 독립적인 주제를 동시에 조사하고 메인이 결과를 모아 정리할 때 유용함.

- 
                <strong>에이전트 팀:</strong>

- 각 팀원은 팀원들끼리 서로 메시지를 주고받을 수 있음.

- 상대방 결과에 반박하며 협업할 수 있는 구조가 가장 큰 특징임.

- 결론적으로 서브 에이전트는 독립적 작업 후 메인 종합, 에이전트 팀은 상호 대화하며 협업하는 방식임.

## **리서치 결과 검토 및 스킬 최적화**

- 
        리서치 완료 후 총 두 가지 파일이 생성됨.

- 'research.md' 파일: 네 개의 주제 리서치 결과를 통합 정리한 문서.

- 'research_sample.html' 파일: 리서치된 모든 패턴이 적용된 샘플 파일.

- 
        생성된 결과 확인.

- 'research.md' 파일에서 순수 HTML, CSS, 자바스크립트로 프레젠테이션을 구현할 수 있도록 통합 결과가 있음을 확인.

- 샘플 파일을 실행하면 다양한 샘플과 함께 방향키로 애니메이션 이동이 가능한 예제 파일이 생성됨을 확인.

- 
        AI 도구 활용 시 중요 강조 사항.

- 샘플 파일이 마음에 든다면 바로 해당 자료를 활용해 PPT 스킬을 생성하면 됨.

- 원하는 결과가 아니라면, 원하는 컨텍스트를 입력하여 원하는 결과가 나올 때까지 수정을 반복해야 함.

- AI 도구가 리서치 작업을 대신 수행하지만, 결과 검토 및 방향 지시는 사용자의 몫임.

- 
        원하는 방향으로 결과 수정 요청.

- 필요하다면 참고 자료 파일이나 원하는 디자인 캡처 이미지를 제공하여 요청할 수 있음.

- 초기 결과는 페이지 안에서 여러 액션이 수행되는 방식이므로, 페이지 단위로 이동하도록 추가 요청함.

- 예시로 10페이지 분량의 샘플 페이지를 생성하고, 페이지 이동 시 한 페이지 단위로 이동하도록 요청함.

- 
        수정 완료된 결과 확인.

- 방향키 이동 시 한 페이지 단위로 잘 이동하는 것을 확인함.

## **스킬 크리에이터를 활용한 웹 프레젠테이션 스킬 생성**

- 
        리서치 자료 준비 및 검토가 끝났으므로, '스킬 크리에이터' 스킬을 활용해 공식 문서 구조에 최적화된 스킬을 생성함.

- 
        스킬 크리에이터에게 "순수 HTML, CSS, 자바스크립트로 웹 프레젠테이션을 만드는 스킬을 생성해 달라"고 요청함.

- 
        추가로 원하는 요청 사항을 구체적으로 기재하고, 아까 수집한 'research.md' 리서치 자료와 'research_sample.html' 샘플 파일을 첨부함.

- 
        참고 자료를 읽고 스킬 생성이 완료됨.

## **생성된 PPT 메이커 스킬 사용**

- 
        생성된 스킬을 활용하여 바로 슬라이드를 만들어 볼 것임.

- 
        슬라이드 제작을 위한 예시 스크립트를 첨부함 (클로드 코드의 작동 원리를 설명하는 스크립트).

- 
        'PPT 메이커' 스킬을 활용해 프레젠테이션 생성을 요청할 것임.

- 
        강의에서 강조한 것처럼, 하나의 문제를 한꺼번에 해결하려고 하기보다 업무를 쪼개어 접근하는 것이 중요함.

- 예를 들어, 40장짜리 PPT를 한 번에 만들었다가 마음에 들지 않으면 수정할 코드가 많아짐.

- 컨펌을 받으며 점진적으로 완성해 나가는 것이 좋음.

- 
        시안으로 세 개를 생성해 달라고 요청함.

- 다크 배경 스타일과 콘텐츠는 중앙 스타일로 지정함.

- 
        시안 생성 완료 및 결과 확인.

- 골드 테마, 시안 테마, 바이올렛 컬러로 웹 PPT가 완성된 것을 확인함.

- 한두 장 정도의 시안을 예상했으나, 모든 스타일이 완성된 것을 확인함.

- 조사한 리서치에 맞게 웹 PPT가 완성된 것을 확인할 수 있음.

- 
        참고로, 이러한 'PPT 메이커' 스킬은 프롬프트 몇 번만 입력하고 만든 것임.

- 
        원하는 애니메이션, 배경, 또는 키보드 `F` 키를 눌렀을 때 전체 화면 전환 기능 등을 추가하고 싶다면, 해당 스킬을 지속적으로 수정하면 됨.

- 
        현재 만든 자료는 강의에서 다운로드받을 수 있도록 업로드해 둘 예정임.

## **스킬을 플러그인 마켓플레이스에 공유**

- 
        만든 스킬을 다른 프로젝트에서 재사용하고 팀원들과 공유하기 위해 플러그인 마켓플레이스를 활용함.

- 이 영상의 핵심은 PPT 스킬을 직접 만들고, 이를 플러그인 마켓플레이스를 통해 직접 공유하는 것을 보여주는 데 있음.

- 
        스킬, 플러그인, 마켓플레이스 개념 정리.

- 
                <strong>스킬:</strong> 쉽게 말해 업무 매뉴얼임. 웹 PPT 스킬처럼 업무 매뉴얼을 만들고 이를 활용해 원하는 프레젠테이션을 만드는 것임. 현재 만든 PPT 스킬은 현재 프로젝트에서만 사용 가능함.

- 
                <strong>플러그인:</strong> 스킬과 같은 클로드 코드의 확장 기능을 다른 곳에서도 쓸 수 있게 묶는 패키지임. 플러그인 안에 스킬, 서브 에이전트, 훅, MCPC 등 다양한 것을 묶어 다른 팀원들과 공유할 수 있음. PPT 스킬 하나만 공유하고 싶다면 해당 스킬을 플러그인으로 공유하면 됨.

- 
                <strong>마켓플레이스:</strong> 이러한 플러그인을 모아 놓은 저장소임. 앱스토어와 같음. 앱스토어에 앱을 등록하면 다른 사람들이 다운로드해 사용할 수 있는 것처럼, 마켓플레이스에 플러그인을 등록하면 다른 사람들이 이를 사용할 수 있음.

- 
        플러그인 제작 과정.

- 'PPT 메이커 플러그인'이라는 플러그인 폴더를 만듦.

- 해당 폴더를 코드 편집기에서 열고 그 안에 '.clode-plugin' 디렉토리를 만듦.

- '.clode-plugin' 디렉토리 안에 'plugin.json' 파일을 만듦.

- 'plugin.json' 파일에는 플러그인 이름, 설명, 버전, 작성자 정보를 기입함. (작성자명은 중괄호 객체 타입으로 작성해야 오류 없음).

- 아까 만든 PPT 스킬 파일을 플러그인 폴더 안의 'skills' 디렉토리로 옮김. (플러그인 폴더 안에 'skills' 디렉토리를 만들고 PPT 메이커 스킬을 붙여넣기 함).

- 이렇게 하면 플러그인이 완성됨.

- 
        마켓플레이스 제작 과정.

- 'My Marketplace'라는 마켓플레이스 폴더를 만듦.

- 마찬가지로 해당 마켓플레이스 안에 '.clode-plugin' 디렉토리를 만들고 그 안에 'marketplace.json' 파일을 만듦.

- 'marketplace.json' 파일에는 마켓플레이스 이름, 소유자 정보, 배포할 플러그인 목록을 넣음.

- 아까 만든 플러그인을 마켓플레이스 폴더 안의 'plugins' 디렉토리에 넣음. (우선 'plugins' 디렉토리를 만들고 그 안에 플러그인을 그대로 넣음. 이름은 맞춰야 함).

- 
        핵심 구조 강조.

- 마켓플레이스 루트의 '.clode-plugin' 안에는 'marketplace.json' 파일이 있어야 함.

- 플러그인 안의 '.clode-plugin' 디렉토리 안에는 'plugin.json' 파일이 있어야 함.

- 
        GitHub에 마켓플레이스 업로드.

- Git 초기화, 커밋 진행 후 GitHub에 게시함.

- GitHub에 업로드된 마켓플레이스를 확인할 수 있음.

- 
        다른 프로젝트에서 마켓플레이스와 플러그인 설치 방법.

- `/plugin marketplace add [GitHub 사용자명]/[리포지토리명]`을 입력하여 마켓플레이스를 추가함.

- `/plugin` 명령어로 우측 '마켓플레이스' 탭으로 이동하면 추가한 'My Marketplace'가 보임.

- 해당 마켓플레이스에 들어가 플러그인 목록을 확인하고, 'PPT Maker Plugin'을 프로젝트 스코프에 설치함.

- 플러그인이 설치된 것을 확인할 수 있으며, '.clode' 디렉토리의 'settings.json' 파일 안에 해당 플러그인이 나타남.

- 클로드 코드를 다시 실행하고 `/ppt maker`를 입력하면 설치한 플러그인 안의 'PPT 메이커' 스킬이 나오는 것을 확인할 수 있음.

- 
        플러그인 시스템의 장점.

- 플러그인으로 만들어 마켓플레이스에 배포해 두면 팀원이 많아도 GitHub 링크 하나만 공유하면 누구나 설치해 사용할 수 있음.

- 업데이트 관리가 편리함. 플러그인 개선 후 GitHub에 푸시만 하면, 플러그인을 설치한 팀원들은 마켓플레이스에서 업데이트만 진행하면 됨.

- 'Enable Auto Update'를 켜두면 클로드 코드 실행 시마다 자동으로 최신 버전을 받아옴. (기본 설정은 꺼져 있으므로 수동으로 켜야 함).

- 스킬만 사용할 경우 업데이트마다 복사 붙여넣기 해야 하지만, 플러그인을 통해 공유하면 번거로운 작업이 사라짐.

- 한 번 잘 만들어 놓으면 어디서든 재사용할 수 있고 자동 업데이트로 편리하게 관리되어 팀 전체의 생산성이 향상됨.

        클로드 코드를 제대로 배우고 싶은 사람들에게 '인프런 클로드코드 완벽 마스터 강의'를 추천함.
