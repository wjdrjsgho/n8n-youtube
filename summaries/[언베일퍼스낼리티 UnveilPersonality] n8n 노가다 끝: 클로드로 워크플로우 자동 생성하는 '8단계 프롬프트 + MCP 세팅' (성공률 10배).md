## **M8N 워크플로우 생성을 위한 AI 활용 방법 및 중요성**

<ol>

- 클로드 AI를 활용한 M8N 워크플로우 생성 방법 소개
    <ol>

- 유튜브에서 클로드 코드를 이용한 M8N 워크플로우 생성 튜토리얼이 많아지고 있음

- 클로드 코드를 사용한다고 해서 워크플로우의 퀄리티가 자동적으로 향상된다고 생각하지 않음

- 클로드 데스크탑과 클로드 코드를 사용하여 M8N 워크플로우를 생성하는 구체적인 방법을 제시함

- 여러 MCP 서버를 활용하고 AI의 사고 범위를 명확히 좁혀 성공률이 높은 방법을 다룸

- M8N보다 안티그래비티나 클로드 코드가 더 좋다는 주장에 대한 견해를 밝힘

- 영상을 통해 AI에게 지시하여 M8N AI 에이전트를 생성하는 단계별 방법과, 인터넷 AI 정보들을 선별하는 방법을 알 수 있음
    </ol>

    <ol>

- **M8N 워크플로우 생성을 위한 MCP 서버스 활용**
        <ol>

- MCP는 AI가 외부 서비스를 직접 조작할 수 있도록 연결하는 통로 역할임 (서비스 구조를 알려주면 AI가 알아서 하도록 오픈해 놓음)

- 총 네 가지 MCP 서버를 활용함
            <ol>

- **컨텍스트 7 MCP 서버**: 각 외부 서비스의 현재 버전에 대한 정확한 문서를 읽게 하여, M8N의 최신 버전을 읽게 함

- **M8N MCP 서버**: AI가 내 M8N을 직접 조작할 수 있도록 함 (워크플로우 생성, 수정, 삭제 권한 부여)

- **M8N 스킬스 서버**: AI가 M8N을 더 잘 쓸 수 있도록 가르치는 가이드 모음임
                <ol>

- M8N 익스프레션, 노드 설정 방법, 에러 처리 방법 등이 포함된 참고서와 같음

- AI가 워크플로우를 만들 때 올바른 방법을 알려주는 역할을 함
                </ol>

- **M8N 워크플로우스 서버**: 기존 M8N 워크플로우 템플릿 4,300여 개를 모아 놓은 서버임
                <ol>

- AI가 기존 템플릿을 참고하여 워크플로우를 만들 수 있게 함

- 카테고리별, 복잡도별, 트리거별로 분류되어 AI가 상황에 맞는 워크플로우를 찾기 용이함
                </ol>

            </ol>

- **M8N API 키 발급**: M8N MCP 서버와 내 M8N이 소통하기 위한 필수 요소임
            <ol>

- M8N 세팅스 > M8N API에서 'create an API key'를 선택하고 'no expiration'으로 발급받음
            </ol>

        </ol>

- **클로드 데스크탑 설정 및 지침 적용**
        <ol>

- **MCP 서버스 파일**: 언웨이 마인드 클럽에서 무료로 제공되는 MCP 설정 정리 파일임
            <ol>

- M8N API URL 부분에 내 API 주소를 복사하여 붙여 넣음

- M8N API 키 부분에 방금 발급받은 API 키를 붙여 넣음
            </ol>

- **클로드 데스크탑 설정 통합**:
            <ol>

- 클로드 데스크탑 설정 > 개발자 구성 편집에서 클로드 데스크탑 콘피그 파일을 열음

- 해당 파일 내용을 AI에게 제공하고, 제공된 MCP 서버스 파일의 내용을 함께 붙여 넣어 두 정보를 통합하도록 AI에게 지시함

- AI가 통합해 준 내용을 복사하여 클로드 데스크탑 콘피그 파일의 내용을 교체함
            </ol>

- **클로드 MD 파일**: AI에게 제공하는 지시 프롬프트 파일임
            <ol>

- 클로드 프로젝트에서 새 프로젝트를 만들고 적절히 이름을 정함

- 프로젝트의 지침 부분에 클로드 MD 파일의 내용을 복사하여 넣음

- 지침 내용은 AI를 M8N 워크플로우 생성 전문가로 정의하고, 사용 가능한 도구 및 생성 순서 등을 명시함
            </ol>

- **M8N 워크플로우 아웃라인 가이드 파일**: 8단계로 구성된 워크플로우 생성 프로세스 가이드임
            <ol>

- AI가 워크플로우를 만들 때 중구난방이 아닌 명확한 생각의 범위를 가지도록 돕는 매뉴얼임

- 프로젝트의 파일 부분에 해당 파일을 넣음
            </ol>

        </ol>

- **클로드 데스크탑을 이용한 워크플로우 생성 예시**
        <ol>

- 간단한 워크플로우 요청: Gmail에서 특정 키워드가 포함된 메일이 오면 노션 데이터베이스에 자동 기록하는 워크플로우를 만들어 달라고 지시함

- 복잡한 워크플로우 요청: AI 뉴스 RSS 피드를 구독하여 매일 아침 9시에 텔레그램으로 보내고, 기사 3개를 골라 한글로 요약 후 중요성을 한 줄로 설명해 달라고 지시함

- 생성된 워크플로우는 오류나 부족한 부분이 있을 경우 직접 수정하거나 대화를 통해 추가 수정이 가능함
        </ol>

    </ol>

- 클로드 코드로 M8N 워크플로우 생성하기
    <ol>

- **클로드 코드의 특징 및 장점 (개발자 관점)**
        <ol>

- 클로드 코드는 일반 클로드 AI와 동일한 LM 모델을 사용하며, 개발자에게 유용한 기능이 추가된 버전임

- 일반인들에게 클로드 코드의 장점은 다음과 같음
            <ol>

- **로컬 파일 접근 가능**: 내 컴퓨터에 저장된 로컬 파일에 AI가 접근할 수 있음 (클로드 코드의 핵심 기능)

- **콘텍스트 제한 비교적 자유로움**: 일반 클로드보다 긴 대화에 대한 콘텍스트 제한이 덜함

- **여러 작업 동시 병렬 처리**: 일반 클로드가 한 번에 한 작업만 하는 반면, 클로드 코드는 여러 작업을 동시에 처리할 수 있음
            </ol>

- AI 에이전트 50개를 동시에 돌린다고 생산성이 50배 오르는 것이 아니며, 정의된 관점과 프로세스가 좋지 않으면 결과물도 좋지 않음

- 굳이 필요 없는데 클로드 코드를 쓸 필요는 없으며, 새로운 AI 툴 공부에 시간을 낭비하기보다 중요한 것에 집중하는 것이 현명함
        </ol>

- **비주얼 스튜디오 코드 설정**
        <ol>

- 비주얼 스튜디오 코드 프로그램을 다운로드하여 설치함

- AI가 로컬 파일에 접근할 수 있도록 내 로컬 디스크에 폴더를 생성함 (예: 'test-n8n')

- 비주얼 스튜디오 코드에서 '파일 > 폴더 열기'를 통해 생성한 폴더를 염

- 익스텐션스에서 '클로드 코드 VS 코드'를 설치함

- 설치 후 오른쪽 위의 클로드 아이콘을 클릭하고 클로드 AI 서브스크립션을 통해 클로드 계정으로 로그인함

- 세팅 > 클로드에서 'dangerously skip permissions'를 체크하여 AI가 매번 허락을 받지 않고 작업을 진행하게 함
        </ol>

- **폴더별 지침 및 파일 적용**
        <ol>

- 각 폴더별로 클로드닷 MD 파일이 지침(시스템 프롬프트) 역할을 함

- 제공된 클로드 MD 파일과 M8N 워크플로우 아웃라인 가이드 파일을 폴더 안에 넣어줌
        </ol>

- **MCP 서버 설정**
        <ol>

- 클로드 코드에서 '바이패스 퍼미션스'를 켜고, M8N MCP가 워크플로우와 에이전트를 생성, 실행, 바로 열 수 있도록 아래 항목에 접근 가능하도록 설정함

- 제공된 MCP 서버스 파일을 클로드 코드에 주고, 첨부된 MCP 서버스를 참고하여 프로젝트 폴더에 MCP를 설정하도록 지시함

- 설정이 완료된 곳에 M8N URL과 API 키를 직접 넣어주면 클로드 AI 데스크탑과 동일하게 설정이 완료됨

- 동일한 LM 모델을 사용하기에 같은 퀄리티의 M8N 워크플로우가 생성됨
        </ol>

- **클로드 코드를 이용한 워크플로우 생성 예시 및 문제 해결**
        <ol>

- '이것 M8N으로 만들어 줘'라고 지시하면 워크플로우가 두 개로 나뉘어 만들어질 수 있음 (클로드 코드의 병렬 작업 능력 때문)

- M8N에 바로 임포트되지 않는 오류가 발생할 수 있으며, 이 경우 직접 복사-붙여넣기하여 워크플로우를 생성함

- 오류 발생 시 AI에게 직접 질문하여 해결책을 찾을 수 있으며, AI가 에디터에서 열리도록 크리셜 직접 연결 등의 해결책을 제시함

- 이 모든 과정에 사용된 지침이나 파일은 언웨이 마인드 클럽에서 무료로 공유함
        </ol>

    </ol>

- AI 정보 필터링 및 M8N 활용에 대한 관점
    <ol>

- **AI 에이전트의 한계**: AI는 명확하게 맞아떨어지는 일은 잘하지만, 알아서 잘 딱 깔끔하고 센스 있게 하는 능력은 부족함

- **만들 수 있다고 다가 아님**: AI로 쉽게 무언가를 만들 수 있어도, 그것이 실제 가치가 있고 운영 가능한지는 완전히 다른 이야기임

- **M8N의 본질과 워드프레스 비유**:
        <ol>

- M8N을 버리고 안티그래비티로 갈아타라는 주장은 외국 유튜버들이 광고비를 받고 하는 말일 수 있음

- AI로 5분 만에 자동화 스크립트를 만들어도 그 구조 파악, 오류 발생 시 원인 분석, 명확한 수정이 어려움

- M8N은 대단히 특별한 AI 도구가 아니라, 운영과 유지보수를 안정적으로 할 수 있게 돕는 '레이어'일 뿐임

- 클릭 몇 번으로 퍼블리시, 스케줄 트리거, 웹 트리거 설정이 가능하며 이미 안정성이 검증되었기 때문에 사용함

- M8N은 워드프레스와 같음
            <ol>

- 워드프레스를 사용하는 이유는 홈페이지 운영이 쉽고 데이터베이스 설정 등을 신경 쓸 필요가 없기 때문임

- AI로 화려한 3D 홈페이지를 만들어도 실제로 운영하려면 복잡해져 워드프레스를 쓰는 것이 나음
            </ol>

- M8N과 워드프레스처럼 안정성이 검증된 것을 버리고 AI가 만들어 준 것을 쓰라는 것은, 실제로 운영을 경험해 보지 않은 사람들의 주장임
        </ol>

- 이 채널에서는 현실에 적용될 수 있는 정보만을 공유할 것임
    </ol>
