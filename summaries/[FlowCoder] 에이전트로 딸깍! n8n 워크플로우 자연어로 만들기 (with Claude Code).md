## **M8 로컬 설치 및 초기 설정**

<ol>

- M8을 로컬에 전역으로 설치하도록 요청함.

- M8 설치 완료를 확인하고, Claude Code가 미리 적어둔 다음 답변을 활용하여 M8을 실행함.

- 잠시 기다리자 로컬에서 서버가 열리고, 기존에 설치된 M8에 로그인하여 정상 설치를 확인함.
</ol>

## **M8N MCP(관리형 노드 패키지) 연동**

<ol>

- M8N MCP 설치를 진행함.
    <ol>

- M8N MCP는 러시아 개발자가 만든 것으로 보이며, 1084개의 M8 노드, 인기 템플릿, 메타데이터 템플릿 등을 포함함.

- 이는 Claude가 종종 가져오는 예전 자료나 잘못된 컨텍스트 정보 대신 최신 정보를 가져오는 데 유용함을 언급함.
    </ol>

- 로컬에 전역으로 설치하기 위해 현재 폴더에서 `git clone`을 통해 MCP를 받도록 요청함.
    <ol>

- Claude Code가 자동으로 M8N MCP를 설치하고 MCP 서버를 추가함.
    </ol>

- M8 API URL을 기존 M8 주소로 설정하고, API 키를 입력함.
    <ol>

- M8 설정에서 `M8N MCP` 메뉴로 이동하여 새로운 키를 발급받음.

- 발급받은 키를 Claude Code에 넣어 세팅하도록 요청함.
    </ol>

- Claude Code를 다시 실행하여 MCP 연결 상태를 확인한 결과, 'Connected' 상태임을 확인함.
</ol>

## **MCP 연동 확인 및 복잡한 워크플로우 구상**

<ol>

- M8 MCP가 정상 작동하는지 확인하기 위해 테스트 워크플로우 생성을 요청함.
    <ol>

- 이 과정에서 Claude 세션 재시작으로 M8이 꺼져 M8을 백그라운드에서 다시 시작함.
    </ol>

- M8에 접속하여 'Test Workflow'가 생성되었는지 확인.
    <ol>

- 생성된 워크플로우는 `Set Data` 노드를 포함하며, ID가 일치함을 확인함.

- M8 MCP 연동을 통해 M8을 직접 제어할 수 있음을 확인.
    </ol>

- 원하는 정보의 뉴스레터를 받는 복잡한 워크플로우 구성을 요청함.
    <ol>

- 매일 아침 '바이브 코딩' 레딧 게시물 중 어제 가장 큰 이슈가 있었던 3개 게시물을 요약하여 이메일로 전달하는 뉴스레터 워크플로우를 설정하도록 요청함.

- 뉴스레터를 받을 이메일 주소를 입력함.
    </ol>

- 워크플로우가 JSON 형태로 생성된 후, MCP에서 검증 및 자동 수정 과정을 거침.
    <ol>

- MCP가 단순히 생성에서 그치지 않고 검증 및 자동 수정하는 과정을 포함하고 있음을 언급함.
    </ol>

</ol>

## **레딧 뉴스레터 워크플로우 상세 구축 및 테스트**

<ol>

- 워크플로우 구조를 확인함.
    <ol>

- 매일 아침 8시 트리거, 레딧에서 '바이브 코딩' 검색, 상위 3개 게시물 필터링, 데이터 통합, 요약 생성, 이메일 전송 단계로 구성됨.
    </ol>

- M8에 필요한 API 키 설정을 진행함.
    <ol>

- <strong>GPT Omnia 크리덴셜 설정:</strong>
        <ol>

- OpenDocs 가이드에 따라 API 키 발급 페이지로 이동함.

- 새로운 테스트 키를 발급받아 복사하고, M8에 '테스트 키'라는 이름으로 저장함.
        </ol>

- <strong>구글 크리덴셜(Gmail) 설정:</strong>
        <ol>

- 구글 클라우드 콘솔에 접속하여 새 프로젝트(예: `0128`)를 생성함.

- API 및 서비스 메뉴에서 'OAuth 동의 화면'으로 이동함.

- 앱 이름, 사용자 지원 이메일, 개발자 연락처 이메일 등을 입력하고 계속 진행함.

- '사용자 인증 정보'에서 'OAuth 클라이언트 만들기'를 선택함.

- 애플리케이션 유형을 '웹 애플리케이션'으로 설정하고, '승인된 리디렉션 URI'를 Claude Code에게 물어봐서 받은 URL을 붙여넣어 생성함.

- 생성된 클라이언트 ID와 클라이언트 시크릿(비밀번호)을 복사하여 M8에 입력함.

- '테스트 사용자' 설정 페이지로 이동하여 테스트 사용자를 추가함.

- M8에서 테스트 사용자로 로그인 과정을 완료함.

- Google Cloud 콘솔에서 'Gmail API'를 검색하여 활성화함.
        </ol>

    </ol>

- 워크플로우를 실행하여 이메일 전송 테스트를 진행함.
    <ol>

- 레딧 검색, 상위 3개 기사 추출, 데이터 합치기 단계를 거침.

- 요약 프롬프트는 "뉴스레터 작성 및 요약, 한국어, 제목, 핵심 내용, 커뮤니티 반응 포함"으로 설정됨.

- 이메일로 뉴스레터가 도착했으나, 마크다운 문법으로 전송되어 디자인이 아쉽다고 판단함.
    </ol>

</ol>

## **이메일 뉴스레터 디자인 개선**

<ol>

- Claude Code에게 "내용이 그대로 전송되어 디자인이 별로야. 개선해 줘"라고 요청함.
    <ol>

- Claude Code는 이메일 템플릿을 HTML 디자인으로 업데이트하겠다고 응답함.

- 업데이트될 템플릿은 HTML 형식, 그라데이션 카드 UI, 헤더/카드/인사이트/푸터 레이아웃으로 구성될 예정임을 밝힘.
    </ol>

- 업데이트된 워크플로우를 다시 실행하여 이메일을 확인.
    <ol>

- 디자인이 개선된 (보라색 계열의) 뉴스레터가 도착했으며, 원문 내용과 링크도 깨지지 않고 잘 포함되어 있음을 확인함.
    </ol>

</ol>

## **레딧 게시물 기반 쇼츠 자동화 워크플로우 구축**

<ol>

- 레딧 게시물을 활용하여 쇼츠 자동화 또는 블로그/SNS 자동화를 위한 워크플로우 생성을 요청함.
    <ol>

- 블라인드 게시물을 가져오는 대신 레딧 게시물을 가져오는 방식이 유용할 수 있다고 언급함.
    </ol>

- MCP가 다시 작동하며, 쇼츠 제작에 사용할 비디오 생성 서비스로 'Creator Weight'를 추천 받아 선택함.
    <ol>

- 이 서비스는 처음 사용해보는 서비스라고 언급함.
    </ol>

- 워크플로우가 매우 빠르게 생성되고 검증됨.

- <strong>쇼츠 워크플로우 구조</strong>를 확인함.
    <ol>

- 트리거, 레딧 '바이브 코딩' 검색, 인기 게시물 1개 선택, 30초 스크립트 생성, TTS 및 이미지 생성, 음성/이미지 병합, 비디오 설정 단계로 구성됨.
    </ol>

- Creator Weight 서비스 가입 후 API 키 발급 및 템플릿 선택을 진행함.
    <ol>

- API 키를 M8에 입력하고, 선택된 템플릿(예: '크리트 템플릿')으로 워크플로우를 업데이트하도록 요청함.

- 초기에는 TTS와 배경음악 관련 노드 수정이 필요함을 확인함.
    </ol>

- 워크플로우 연결 상태가 바르지 못하다고 판단하여 연결 확인 및 수정을 요청함.
    <ol>

- Claude Code가 문제를 찾아내고 연결을 수정하여 워크플로우가 정상적으로 작동함을 확인함.
    </ol>

- 워크플로우를 실행하여 쇼츠가 빠르게 생성되는 것을 확인함.
    <ol>

- 생성 완료 시 이메일로 알림이 발송됨 (디스코드 알림으로도 대체 가능하다고 언급됨).
    </ol>

</ol>

## **결론 및 향후 계획**

<ol>

- M8을 에이전트처럼 사용하여 워크플로우를 구축하는 것이 매우 효과적임을 확인함.

- 수동으로 M8을 가르치거나 바이브 코딩 서비스와 결합하는 방법에 대한 고민이 불필요함을 느낌.

- 향후 이 과정을 더욱 자동화하거나 Claude에게 맡기는 방식을 변경하는 등의 가능성을 언급함.

- M8을 넘어서 '에이전틱 코딩'이라는 새로운 코딩 방법들을 계속 소개할 예정임을 밝힘.
</ol>
