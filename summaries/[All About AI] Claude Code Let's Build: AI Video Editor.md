## **AI 비디오 에디터 프로젝트 구상 및 개발 목표**

- 주말 동안 여가 시간이 많아 클라우드 코드(Cloud Code)를 활용한 프로젝트를 진행해보기로 함.

- AI 비디오 에디터를 개발하는 것이 목표로, 반자동(semi-autonomous) 영상 편집기를 만드는 작업임.

- 주요 내용은 클라우드 코드를 사용하여 아이디어를 현실화하고, 그 워크플로우를 보여주는 데 중점둠.

## **프로젝트 필수 도구 및 기술 스택 준비**

- 시작 전에 필요한 도구는 다음과 같음.

- <strong>FFmpeg</strong>: 영상 및 오디오 처리를 위한 필수 도구.

- <strong>Whisper 모델</strong>: 오디오를 텍스트로 변환하기 위해 로컬에서 실행할 모델 필요함.

- <strong>Gemini 3 Flash API</strong>: 빠르고 성능이 좋으며 함수 호출(function calling)이 가능한 API로 선정함.

- 추가 기능 고려 사항으로 Nano Banana Pro나 다른 비디오 모델을 사용할 수도 있으나, 우선 위 도구들로 시작할 예정임.

- 개발은 처음부터 끝까지 진행할 것임.

- 시연의 편의를 위해 Mac 터미널을 사용하지만, ID 내에서도 동일하게 사용 가능함.

## **초기 개발 환경 설정 및 문서 통합**

- 먼저 `AI video editor`라는 디렉토리를 생성하고 Cursor ID에서 해당 디렉토리를 열음.

- Gemini 3 API 문서 가져오기:

- `Gemini API/docs` 페이지에서 Gemini 3 문서를 마크다운 형식으로 가져옴.

- `docs` 폴더를 만들고 그 안에 `gemini_tree.md` 파일로 저장함.

- 모델이 작업을 수행할 수 있도록 함수 호출(function calling)이 필요하여, 관련 문서도 `function_call.md` 파일로 저장함.

- FFmpeg 설치 확인: 설치되지 않았다면 `ffmpeg.org`에서 Linux, Windows, Mac OS용으로 다운로드해야 함.

- Whisper 모델 준비: OpenAI GitHub (`github.com/openai/whisper`)에서 로컬에서 오디오를 텍스트로 전사하는 데 사용할 모델을 확보함.

- API 키 설정:

- `.env` 파일을 생성하고 `Gemini API key`를 추가함.

- Google에서 API 키를 가져와 설정함 (OpenAI, Anthropic, XAI 등 다른 모델도 `.env` 파일 설정으로 사용 가능).

## **Cloud Code를 활용한 프로젝트 계획 수립**

- AI 에디터의 사용자 인터페이스(UI)는 간단하게 구성할 예정임.

- 기술 스택은 Vercel의 Next.js와 JavaScript를 기반으로 할 것임.

- Cloud Code를 활용하여 프로젝트를 구체화함.

- `plan` 모드로 설정함.

- 준비된 문서(Gemini API 문서 등)와 `.env` 파일에 저장된 API 키 정보를 Cloud Code에 제공함.

- 음성-텍스트 변환(speech to text) 기능을 사용하여 AI 비디오 에디터의 첫 번째 초안(draft)에 대한 상세한 프로젝트 계획을 요청하는 긴 프롬프트를 입력함.

- Cloud Code가 계획 수립 과정에서 몇 가지 추가 질문을 함.

- 사용할 Whisper 모델 종류: `base` 또는 `small` 중에서 `small` 모델이 효율적일 것 같아 선택함.

- 타임라인에 비디오 파형 시각화 포함 여부: 일단 간단한 타임라인으로 시작하도록 함.

- 파일 업로드를 위한 드래그 앤 드롭 기능 여부: 이 기능을 추가하기로 함.

- Cloud Code가 제시한 계획을 승인함. 계획에는 Next.js, 업로더/비디오 플레이어/타임라인/클립 에디터 컴포넌트, FFmpeg 통합, Whisper 기반 오디오 추출 파이프라인, Gemini 3.5 Flash 클라이언트를 활용한 AI 모드 등이 포함되어 있음.

- 계획 승인 후, Cloud Code가 앱 빌드를 시작함.

## **기본 편집기 기능 구현 및 테스트**

- Cloud Code가 앱을 빌드하는 데 약 6분 소요됨.

- 테스트용 비디오로 사무실에서 Claude가 사업을 운영하는 내용의 `Claude business` 파일을 준비함.

- `npm install`이 완료되고, `npm run dev`를 실행하여 첫 번째 이터레이션을 테스트함 (포트 3001).

- 기본 기능 테스트 결과:

- 요청했던 드래그 앤 드롭 기능을 통해 `Claude business` 비디오를 성공적으로 업로드함.

- 비디오가 타임라인에 나타남.

- 타임라인 조작 지침을 확인: 타임라인 클릭하여 탐색, 'C' 키로 재생 헤드에서 자르기, 클립 선택 후 'Delete' 키로 삭제.

- 자르기 기능: 타임라인을 스크롤하여 30초 구간을 자르는 시도를 하니 직관적으로 잘 작동함.

- 삭제 기능: 처음에는 'Delete' 키가 작동하지 않았으나, 다시 시도하자 잘 작동하여 잘라낸 클립을 삭제할 수 있었음.

- 클립 합치기/재정렬 문제 발생: 클립을 자르고 삭제하자 남은 클립들이 분리되어 있어 합칠 방법이 필요하다는 것을 인지함.

- Cloud Code에 피드백 및 업데이트 요청:

- 현재 상태를 스크린샷으로 캡처하여 Cloud Code에 제공함.

- 타임라인에서 일부를 잘라냈지만, 남은 클립들을 병합할 수 있는 기능이 필요하다고 설명함.

- 오른쪽에 있는 클립을 왼쪽으로 옮겨 이전 클립과 연결하고, '병합' 또는 '삭제' 클릭 시 자동으로 병합되어 원본보다 짧은 클립이 되도록 업데이트를 요청함.

- 업데이트된 기능 재확인:

- Cloud Code가 변경 사항을 적용한 후, 비디오 업로드, 자르기, 불필요한 클립 삭제, 드래그를 통한 순서 변경 기능이 추가되었음을 확인함.

- `Claude business` 비디오를 다시 드롭함.

- 1분 지점에서 30초를 자르고 삭제하자, 클립들이 자동으로 다시 합쳐져 타임라인 길이가 540초로 줄어든 것을 확인함.

- `Export clips` 버튼을 클릭하여 비디오를 내보냄.

- 내보낸 비디오를 다운로드하여 재생해보니, 편집된 내용(Claude 관련 특정 대화 부분이 제거된)이 잘 반영되어 있었음.

- 기본적인 비디오 편집기 기능(자르고 붙이기)이 성공적으로 작동하는 것을 확인함.

## **AI 편집 기능 구현 및 테스트**

- AI 편집 기능 테스트 준비:

- 동일한 `Claude business` 비디오를 다시 업로드함.

- `AI edit` 버튼을 클릭한 후, Whisper를 사용하는 `Start transcription`을 선택함.

- 비디오는 MP3로 변환되고 Whisper를 통해 텍스트로 전사됨.

- 전사된 텍스트(74개 세그먼트)를 확인함.

- 첫 번째 AI 편집 프롬프트 테스트: "Claude가 '미쳐버리는'(goes kind of off the rail) 부분만 남겨줘. 단, 10초 이상 클립만."

- 결과: 10초 이상 클립 두 개가 생성되었으나, AI가 "미쳐버리는" 부분의 기준을 의도와 다르게 해석한 듯 보임.

- 두 번째 AI 편집 프롬프트 테스트: "Claude가 '미쳐버리고'(goes off the rail) '정신 나가는'(goes crazy) 부분만 남겨줘."

- 결과: 6초, 33초, 105초 길이의 클립 세 개가 생성됨.

- 내보내기 후 재생하여 확인 결과, 클립 내용이 Claude가 인간에게 속거나, 정체성 위기를 겪거나, 비현실적인 주장을 하는 등 '미쳐버리는' 상황과 잘 일치하는 것을 확인.

- 이 프롬프트에서는 AI 편집 기능이 잘 작동함을 확인.

- 세 번째 AI 편집 프롬프트 테스트: "Claude가 돈을 벌었는지 안 벌었는지에 대한 부분만 남겨줘."

- 결과: 15초 길이의 클립 한 개가 생성됨.

- 내보내기 후 재생하여 확인 결과, 클립 내용이 Claude가 손실을 줄이고 소량의 돈을 벌었다는 내용과 정확히 일치.

- 자연어 프롬프트만으로 원하는 내용의 클립을 추출하는 AI 편집 기능이 성공적으로 작동하는 것을 확인함.
