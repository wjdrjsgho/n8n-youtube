- 클로드 코드의 플러그인과 애드온 생태계가 매우 빠르게 성장 중임.

- 새로운 MCP, CLI, 스킬, 프레임워크 등이 매일 쏟아져 나와 유용한 것과 아닌 것을 구별하기 불가능하게 느껴짐.

- 생산성 향상을 위해 실제 사용해야 할 도구를 소개하고자 함.

- 이 영상에서 추천하는 클로드 코드 플러그인 및 CLI 도구 Top 10을 제시함.

- 이 도구들은 직접 사용하고 있으며, 진정한 가치를 제공한다고 믿음.

- 각 도구에 대해 다음 네 가지를 알려 줄 것임:

- 이 도구에 관심을 가져야 하는 이유

- 사용 방법

- 실제 사용 사례

- 일부 도구는 익숙할 수도 있고, 이미 다룬 적 있는 것도 있으나, 최소한 몇 개는 처음 접하길 바람.

- 최소 하나의 새로운 도구를 스택에 추가한다면 이 영상이 충분히 가치 있다고 생각함.

## **도구 1: Supabase MCP 서버 (그리고 CLI 전환의 중요성)**

- <strong>Supabase MCP 서버에 관심을 가져야 하는 이유</strong>

- 데이터베이스 및 인증 기능 제공: 개발 시 데이터베이스와 인증이 필요한 프로젝트가 많음.

- 넉넉한 무료 티어: 비용 없이도 상당한 수준까지 사용 가능함.

- 오픈소스 기반: PostgreSQL, Redis 등 오픈소스 도구 위에 구축되어 있음.

- 쉬운 사용: Supabase는 이를 쉽게 사용할 수 있게 감싸주는 래퍼 역할만 함.

- 셀프 호스팅 및 로컬 개발: 인증 및 데이터베이스 프레임워크 전체를 셀프 호스팅하고 로컬에서 개발할 수도 있음.

- <strong>설치 방법</strong>

- 매우 간단함: 한 줄의 코드만 필요함.

- 위치: 클로드 코드 문서의 MCP 섹션에서 찾을 수 있음.

- <strong>사용 사례</strong>

- 데이터베이스가 필요할 때: 기본적인 PostgreSQL 데이터베이스든 Redis 같은 벡터 데이터베이스든 사용 가능함.

- 인증이 필요할 때: 사용자가 사이트를 방문하거나, 프로필을 만들어야 하거나, 데이터를 분리 보관해야 할 때 완벽한 사용 사례임.

- <strong>클로드 코드 내 실제 사용 방법</strong>

- 다른 MCP와 마찬가지로 설치 명령어 실행 후 'MCP' 입력하면 됨.

- 프로젝트 섹션 아래에 'Supabase MCP'가 나타날 것임.

- 초기 인증 필요: Supabase에서 API 키를 받아와야 함. 클로드 코드가 단계별로 안내해 줄 것임.

- 자연어 사용: 연결 후에는 더 이상 Supabase에 들어가 SQL 코드를 작성할 필요 없이 자연어로 데이터베이스를 만들거나 편집 가능함.

- <strong>Supabase CLI로의 전환 설명 (중요한 이유)</strong>

- 사실 Supabase MCP 서버는 실제로 사용할 필요가 없음. 의미가 없는 이유는 대신 Supabase CLI를 사용해야 하기 때문임.

- <strong>시간 낭비의 두 가지 이유</strong>
                <ol>

- <strong>개인적 경험 공유</strong>: 하루 종일 이런 도구를 살펴보는 사람(필자)도 최근까지 Supabase MCP를 사용했음. 다른 영상의 댓글을 보기 전까지는 Supabase CLI의 존재조차 몰랐음. CLI 도구가 MCP 도구보다 우수하다는 것을 알고 있었음에도 깨닫지 못했음. 이 분야의 전문가도 자주 사용하는 도구조차 따라가기 힘들다는 것을 보여주고자 함. 압도당하는 느낌이 들어도 괜찮음.

- <strong>CLI 대 MCP 논의의 가치</strong>: 나머지 9개 도구 중 MCP가 몇 개 있긴 하지만, 대부분의 경우 항상 MCP 대신 CLI를 사용해야 함.

- MCP는 기본적으로 CLI 도구가 하는 것을 흉내 내려는 것에 불과하지만, 터미널 안에 있지 않아 추가적인 오버헤드가 발생함.

## **도구 1 (재정의): Supabase CLI**

- <strong>사용 이유</strong>: Supabase MCP와 동일하게 인증 및 데이터베이스를 다룸.

- <strong>사용 방법</strong>: 거의 같음.

- <strong>가장 큰 차이점: 설치</strong>

- Supabase 문서에 따르면 설치 방법은 운영 체제에 따라 다름.

- 클로드 코드가 이 모든 것을 처리해 줄 수 있음: "Supabase CLI 도구를 다운로드하고 설치해 줘"라고 말하면 됨.

- 해당 의존성을 다운로드하고 설치해 줄 것임. 이제 Supabase CLI에 접근 가능함.

- <strong>중요 사항: 스킬 추가</strong>

- 클로드 코드에 새로운 CLI 도구를 줄 때는 보통 적절한 스킬도 함께 추가하는 것이 좋음.

- 특정 CLI 도구를 특정 방식으로 사용하는 방법을 클로드 코드에 알려주고 싶은 것임.

- 이를 위한 완벽한 방법이 바로 스킬임.

- 새로운 CLI를 설치할 때마다 GitHub이나 문서를 확인하여 이 도구를 위해 만들어진 스킬이 이미 있는지 확인해야 함.

- Supabase는 클로드 코드 플러그인으로 스킬을 제공하며, 마켓플레이스에서 바로 찾을 수 있음.

## **도구 2: Anthropic의 Skill Creator 스킬**

- <strong>강력함</strong>: 오늘 다루는 10개 도구 중 가장 강력한 도구로 생각됨.

- <strong>기능</strong>: 단순히 스킬을 만드는 것을 넘어, 기존 스킬을 수정하고 개선하며 성능을 측정할 수 있음.

- <strong>중요성</strong>: Anthropic이 이 스킬의 개선 사항에 대해 블로그 전체를 할애해 설명할 정도로 대단한 일임.

- <strong>개선 사항</strong>

- 제대로 작동하는 커스텀 스킬 생성 가능: 이전에는 스킬의 효과가 무작위적이었고 실제 데이터가 없었음.

- A/B 테스트 가능: 스킬을 적용했을 때와 안 했을 때, 현재 버전과 이전 버전을 비교하여 실제로 더 잘 작동하는지 측정 가능함. 이전에는 감에 의존했으나 이제는 아님.

- 클로드 코드 개선에 극도로 중요함.

- <strong>설치 방법</strong>

- 매우 쉬움: '플러그인'을 입력한 후 플러그인 목록에서 '스킬 크리에이터'를 검색하고 설치를 누르면 됨.

- 설치 후 '스킬 크리에이터'로 표시될 것임.

- <strong>사용 방법</strong>

- '스킬 크리에이터'를 입력하면 바로 실행됨.

- 자연어 사용: "클로드 코드야, 스킬 크리에이터를 사용해서 새 스킬을 만들거나 현재 스킬을 평가해 줘"라고 말하면 됨.

- <strong>내부 작동 방식</strong>: 다른 스킬과 마찬가지로 내부적으로는 텍스트 프롬프트임. Anthropic의 GitHub에서 클로드 코드의 스킬 섹션을 보면 전체 내용을 볼 수 있음.

## **도구 3: GSD (Get Stuff Done) 프레임워크**

- <strong>성격</strong>: 클로드 코드 위에 올라가는 오케스트레이션 레이어로, 새 프로젝트 생성 시 클로드 코드의 작동 방식을 변경함.

- <strong>주요 기능</strong>: 스펙 기반 개발에 대한 가드레일(guardrails)을 더 많이 제공함.

- <strong>적합한 경우</strong>: 처음부터 새 프로젝트를 만들면서 단계별, 기능별로 체계적으로 나누고 싶을 때 완벽함.

- <strong>컨텍스트 관리</strong>: 컨텍스트 부패 관리, 컨텍스트 윈도우 관리를 매우 잘함.

- 계획의 새로운 부분을 실행할 때마다 새로운 컨텍스트 윈도우를 사용하고 서브 에이전트를 활용함.

- 윈도우 후반부에 진입하여 출력 품질이 떨어지는 일이 없도록 모든 것이 설계되어 있음.

- <strong>권장</strong>: 프로젝트를 처음부터 매우 체계적으로 만들어 줄 무언가를 찾는다면 GSD를 강력히 추천함.

- <strong>설치 방법</strong>

- 간단함: GitHub에서 'GSD' 또는 'Get Stuff Done'을 검색하면 한 줄 명령으로 설치 가능함.

- <strong>사용 방법</strong>

- 새 프로젝트 시작은 모두 슬래시 명령어('JSB 뉴 프로젝트')로 함.

- 기본적으로 스테로이드를 맞은 플랜 모드가 시작됨.

- 만들려고 하는 것을 정말 꼼꼼하게 파악하려고 함. 양질의 결과물을 얻으려면 클로드 코드와 300번 계획을 세워야 하는 경험을 해봤다면 그 필요성을 알 것임.

## **도구 4: Noteable CLI & 스킬**

- <strong>역할</strong>: 요즘 클로드 코드 리서치 워크플로우의 핵심으로, Noteable을 클로드 코드에 연결해 주는 도구임.

- <strong>기능</strong>: 공식 API가 없음에도 Noteable에서 할 수 있는 거의 모든 것을 클로드 코드 터미널에서 가능하게 함.

- <strong>산출물 생성</strong>: 리서치, 분석, 영상, 인포그래픽, 슬라이드덱, 플래시카드, 팟캐스트 같은 산출물 생성을 거의 무료로 Noteable에 맡길 수 있음. 엄청난 가성비임.

- <strong>설치 방법</strong>

- 매우 간단함: Noteable GitHub에 가서 명령어를 복사하고 터미널에 붙여넣어 의존성을 설치하면 됨.

- 의존성 설치 후 로그인 필요: 터미널에서 '노트북룸'을 실행하면 브라우저가 팝업되고 Noteable 계정으로 로그인하면 됨.

- <strong>스킬 추가</strong>: CLI 도구이므로 스킬이 필요함. 스킬 설치 방법도 제공되며, Noteable 스킬을 설치하면 클로드 코드에게 이 모든 것의 사용법을 알려주는 것임.

- <strong>사용 방법</strong>: 실제 명령어는 있지만, 여러분은 그냥 자연어로 클로드 코드와 대화하면 됨. Noteable에서 할 수 있는 모든 것을 터미널을 통해 클로드 코드에서도 자연어만 사용하면 가능함.

- <strong>권장</strong>: 가장 좋아하는 것 중 하나이니 꼭 스택에 추가하길 바람.

## **도구 5: Obsidian**

- <strong>적합 사용자</strong>: 클로드 코드를 개인 비서로 사용하는 분들께 완벽함.

- <strong>가치</strong>: Obsidian은 텍스트 파일들이 어떻게 연결되는지 보여주고 훨씬 체계적으로 관리해 주므로 최종 사용자에게 매우 유용함.

- <strong>사용 사례 (개인 비서)</strong>: 일반적인 코딩 프로젝트에서는 그런 컨텍스트가 맞지 않아 큰 가치가 없음. 하지만 텍스트 문서와 마크다운 파일 형태로 방대하고 계속 쌓이는 컨텍스트가 있는 개인 비서 환경에서는 클로드 코드와 Obsidian 조합이 정말 좋음.

- <strong>설치 방법</strong>: 설치할 CLI도 없고 스킬도 사실 필요 없음.

- Obsidian을 다운로드하고 볼트로 지정할 폴더를 만듦.

- 그 폴더에서 터미널로 클로드 코드를 열기만 하면 됨. (예: 필자의 경우 폴더 이름이 '더 볼트'임.)

- <strong>사용 방법</strong>: 클로드 코드에게 마크다운 파일을 만들 때 Obsidian 규칙을 따르라고 말하면 됨.

- <strong>제한</strong>: 이 연결의 가치는 특정 사용 사례에 한정되며, 만능은 아님.

## **도구 6: Vercel CLI**

- <strong>목적</strong>: 배포 관리를 더 쉽게 해줌.

- <strong>선호 이유</strong>: 배포 시 Vercel을 즐겨 사용하는데, 간단하고 무료 티어도 훌륭하기 때문임.

- <strong>설치 방법</strong>

- 매우 간단함: 단지 이 명령어만 입력하면 됨.

- 클로드 코드에게 "버셀 크리를 설치해 줘"라고 말해도 됨. 이제 패턴을 알 것임.

- <strong>스킬 추가</strong>: 도구를 추가했으니 스킬이 필요함. Vercel이 공식 웹사이트에서 제공하고 있음.

- <strong>고급 사용</strong>: 새로 나온 에이전트 루프와 함께 사용하기에 정말 좋은 도구임. 프로젝트를 변경할 때마다 계속 배포한다면 Vercel CLI 스킬을 호출해서 배포 상태를 확인하는 루프를 만들 수 있음.

## **도구 7: Playwright CLI**

- <strong>기능 강화</strong>: 클로드 코드의 브라우저 자동화 능력을 강화함. 웹에 나가서 실제로 뭔가를 하는 능력, 예를 들어 아마존에 가서 장바구니에 담고 구매하는 것 같은 단순한 것도 가능함.

- <strong>출처</strong>: Microsoft의 오픈소스 도구로 매우 인기 있음.

- <strong>경험</strong>: 브라우저 자동화와 클로드 코드에서 사용하며 정말 즐겼음.

- <strong>설치 방법</strong>

- 매우 쉬움: Playwright CLI GitHub에 가서 명령어를 복사하여 터미널에 붙여넣으면 의존성이 설치됨.

- <strong>스킬 설치</strong>: "플레이라클라이 인스톨 스킬즈"만 입력하면 설치 완료됨.

- <strong>사용 사례</strong>

- 클로드 코드에서의 Playwright 딥 다이브 영상을 만들었으니 확인 바람.

- 기본적으로 브라우저에서 할 수 있는 모든 것을 Playwright로 할 수 있음.

- UI 디자인 테스트나 폼 제출 테스트 같은 것들.

- 프로젝트나 앱을 변경할 때 수동으로 테스트해야 하는 번거로운 작업들을 Playwright가 대신해 줌.

- <strong>사용 방법</strong>: 다른 CLI 도구들처럼 자연어로 작업을 실행하면 됨.

- <strong>특징</strong>: 딥 다이브 영상에서 언급하지 않은 것 중 하나가 `playwright cr show` 명령어임. 이는 헤드리스로 실행 중이더라도 모든 브라우저 에이전트가 뭘 하고 있는지 보여주는 시각적 대시보드를 제공함. 정말 멋진 기능임.

## **도구 8: GitHub CLI**

- <strong>친숙한 플랫폼</strong>: 여러분에게 친숙한 플랫폼과 관련된 것임.

- <strong>기능</strong>: GitHub에서 수동으로 하던 모든 것을 CLI로 할 수 있음.

- <strong>시너지</strong>: Vercel CLI 도구와 함께 사용하면 좋음. 코딩부터 GitHub 푸시, 배포까지 모든 것을 한 곳에서 자연어로 할 수 있음.

- <strong>설치 방법</strong>: 당연히 클로드 코드가 안내해 줄 수 있지만, GitHub 페이지에 운영 체제별로 설명되어 있어 혼란스러울 일은 없을 것임.

- <strong>스킬 필요성</strong>: 클로드 코드 같은 코딩 에이전트는 Git과 GitHub에 이미 능숙해서 이를 완전히 활용하는 데 스킬이 필요 없음. 하지만 원한다면 스킬 크리에이터로 직접 만들면 됨. 솔직히 도구만 설치하고 클로드 코드에게 자연어로 말하는 것만으로도 충분함.

## **도구 9: Firecrawl CLI & 스킬**

- <strong>목적</strong>: 웹 스크래핑에 관한 것임. Firecrawl CLI와 스킬을 사용할 것임.

- <strong>설치 방법</strong>

- 쉬움: 'Firecrawl CLI' 도구를 검색하면 이 페이지가 나옴.

- 한 줄 명령으로 적절한 스킬 설치도 포함되어 있음.

- <strong>특징</strong>: Firecrawl을 처음 접하는 분들을 위해 설명하자면, 웹 스크래핑인데 가져오는 데이터가 AI 에이전트에 맞춤 제작되어 있음.

- <strong>핵심 명령어</strong>: `scrape`, `crawl`, `map`, `search` 네 가지임.

- <strong>사용 방법</strong>: 웹 스크래핑이 처음이라면 이 네 가지 명령어가 다 같은 일을 하는 것처럼 보일 수 있음. 하지만 다행히도 상황에 맞는 명령어를 쓸 줄 아는 존재가 있음. 바로 클로드 코드임. 그래서 스킬이 필요한 것임.

- <strong>사용 사례</strong>: 경쟁사 리서치, 문서 사이트 변경 모니터링, 소스 기반 심층 리서치 같은 표준 사용 사례를 제공함. 정말 좋아하는 도구임. 클로드 코드의 웹 검색 도구를 사용하는 것보다 확실히 한 단계 위임.

## **도구 10: Excalidraw Diagram 스킬**

- <strong>개인적 선호</strong>: 개인적으로 가장 좋아하는 것 중 하나임. 최근에 사용하기 시작했음.

- <strong>감사 인사</strong>: 만든 사람인 콜레이딘에게 감사 인사를 전함. 유튜브에서 그의 채널을 봤을 것이며, 이 분야의 대가 중 한 분임. 그의 작업도 좋아하지만, 이 스킬이 더 좋음.

- <strong>기능</strong>: 클로드 코드에서 자연어로 다이어그램을 만들 수 있게 해줌.

- <strong>활용 사례</strong>: 마지막 영상인 '클로드 코드의 6단계'를 보셨다면 이런 다이어그램들을 보았을 것임. 이 모든 것을 수동으로 한 것이 아니라, 전부 클로드 코드로 만들어 시간을 엄청 절약했고 발표할 자료도 생겼음. 프레젠테이션을 하거나 Excalidraw를 사용해 본 적이 있다면 정말 대단한 것임. 수작업으로 하면 정말 고되지만, 이 스킬이 그 모든 수작업을 없애 줌.

- <strong>설치 방법</strong>: Excalidraw Diagram 스킬 GitHub에서 레포를 클론하고 프로젝트의 스킬 디렉토리에 복사하면 됨.

- <strong>사용 방법</strong>: 다른 모든 도구와 마찬가지로 사용할 때는 클로드 코드에게 자연어로 말하면 됨.

- <strong>가장 좋은 사용법</strong>: 다이어그램을 프레젠테이션의 보조 요소로 사용하되, 클로드 코드에게 지식의 디렉토리 같은 것을 가리키는 것임. 예를 들어, 클로드 코드의 6단계에 대해 이야기할 내용 전체를 정리한 것을 주고, 그 정보를 전부 가져와 각 단계를 차트로 만들어 달라고 요청하는 것임. 물론 플랜 모드에서 좀 주고받으면서 어떤 결과가 나올지 어느 정도 감을 잡았지만, 정말 대단함. 이 도구를 아무리 칭찬해도 모자람.
