## **반복 웹 업무 자동화의 어려움과 Record & Replay 기능 소개**

- 기존 반복 웹 업무 자동화의 한계

- 웹상에서 직접 클릭해야 하는 업무는 자동화가 어려웠음.

- 공식 API나 RPA (Robotic Process Automation) 같은 것이 제공되지 않는 서비스의 경우 더욱 난이도가 높음.

- AI에게 일일이 요청하는 방식은 느리고 번거로워 수동 작업으로 처리하는 경우가 많았음.

- 자동화 스크립트 작성 및 매크로 구현은 비개발자에게 안정적으로 작동시키기 번거로운 일임.

- 코덱스 Record & Replay 기능 출시

- 위와 같은 문제 해결을 위해 코덱스가 'Record & Replay' 기능을 출시했음.

- 기능 설명: 사용자 행동을 한번 녹화하여 보여주기만 하면, 코덱스가 이를 학습하여 재사용 가능한 스킬로 즉시 만들어 주는 기능임.

- 주요 활용 사례:

- 매달/매주 사이트에서 리포트 다운로드 및 정리 작업.

- 온라인 폼 제출 작업.

- 파일 업로드 작업 등 반복적인 웹 기반 업무.

- 본 기능은 반복 업무를 효율적으로 자동화하는 데 매우 유용할 수 있음.

## **Record & Replay 기능의 원리 및 특징**

- 신입 직원 온보딩 방식에 비유

- 신입사원에게 매뉴얼만 제공하기보다, OJT(On-the-Job Training) 형태로 옆에서 직접 시현하며 업무를 가르치는 방식과 유사함.

- 여기서 코덱스는 '신입 직원' 역할이라고 생각하면 됨.

- 작동 방식

- 사용자가 화면에서 업무 처리 과정을 한번 시현함.

- 코덱스가 시현된 과정을 보고 스스로 매뉴얼을 작성함.

- 코덱스는 이 매뉴얼을 '스킬'이라고 부름.

- 스킬이 생성되면, 이후 코덱스가 해당 스킬에 따라 작업을 자동으로 진행함.

- 스킬의 신뢰성 및 개선 필요성

- 코덱스가 생성한 스킬 초안을 100% 신뢰할 수는 없음.

- 대부분의 경우 한 번에 잘 제작되지만, 사용자의 시현이 매끄럽지 않거나 프로세스가 복잡하면 스킬 문서가 다소 애매하게 작성될 수 있음.

- 따라서 스킬 제작 완료 후에는 항상 해당 문서를 검토하여 개선할 부분이 없는지 확인하는 것을 권장함.

## **Record & Replay 기능 사용 준비 및 설정**

- 코덱스 앱 설치

- 현재 이 기능은 신규 기능이므로 macOS 버전의 코덱스에서만 먼저 테스트할 수 있음. (Windows 사용자는 추후 적용 시 활용 가능)

- 컴퓨터 유즈 기능 활성화

- 플러그인 메뉴에서 'Computer Use'를 찾아 설치함.

- 코덱스 설정(Settings)의 'Computer Use' 섹션에서 크롬(Chrome)이나 다른 앱을 조작할 수 있도록 활성화함.

- 만약 크롬이 목록에 뜨지 않으면, 플러그인에서 'Chrome'을 검색하여 별도 플러그인을 설치해야 함.

- Record & Replay 플러그인 설치

- 왼쪽 플러그인 메뉴에서 'record'를 검색하여 'Record & Replay' 플러그인을 선택하고 설치함.

- 설치 완료 후 바로 활용할 수 있음.

## **Record & Replay 활용 예시 1: 월간 매출 리포트 자동화**

- 자동화 시나리오: 구식 컴퍼니 월간 매출 대시보드에서 CSV 다운로드, 파일 정리, 월간 매출 리포트 생성

- 테스트 환경

- 테스트용으로 가안의 '구식 컴퍼니 월간 매출 대시보드' 사이트를 생성했음.

- 이 사이트에서는 월별 데이터 확인 및 CSV 다운로드 버튼이 제공됨.

- 일반적으로 추가 분석을 위해 CSV 파일을 다운로드하고 리포트를 생성하는 작업에 활용됨.

- 폴더 구조 설정

- CSV 다운로드 파일은 원래 다운로드 폴더에 저장되지만, 이를 특정 구조로 정리하고자 함.

- `로우` 폴더를 생성하고, 그 안에 월 단위로 `YYYYMM` 형식의 폴더를 만들어 CSV 파일을 저장함. (예: `로우/202605/`)

- `리포트` 폴더를 생성하고, 그 안에 최종 리포트 파일을 저장함.

- Record & Replay를 이용한 자동화 과정 (시현)

- 주로 작업하는 90 프로젝트 폴더로 설정함.

- `@record` 명령어를 입력하여 Record & Replay 플러그인을 실행함.

- 작업 내용 프롬프트 설명: "매출 대시보드에서 선택한 달의 매출을 내려받아 정해진 폴더로 옮기고 월간 매출 리포트를 만들고 싶다"고 설명함.

- 코덱스의 녹화 요청에 'Allow' 버튼을 눌러 녹화를 시작함.

- 실제 작업 시현:

- 웹사이트에 접속함.

- 월별 선택기에서 '2026년 5월'을 선택함.

- 데이터가 뜨면 'CSV 다운로드' 버튼을 클릭함.

- 다운로드된 CSV 파일을 `로우/202605` 폴더를 생성하고 그 안에 이동시킴.

- `리포트` 폴더에 `202605 세일즈 리포트.md` 파일을 생성하고 '리포트가 들어갈 자리'라고 내용을 입력함.

- 작업 완료 후 녹화를 중지함.

- 코덱스의 스킬 생성 및 분석

- 코덱스가 녹화된 작업을 분석하여 핵심 흐름을 파악함.

- 파악된 흐름: "크롬에서 월 연도 선택 -> CSV 다운로드 -> 로우 폴더 생성 -> 파일 배치 -> 리포트에 마크다운 생성"

- 이를 반복 실행 가능한 '스킬' 파일(스크립트 기반)로 생성함. 스킬 이름은 '세일즈 리포트'로 생성되었음.

- 생성된 스킬 파일에는 인풋 포맷, 워크플로우 설명, 결과물 검증 (`베리파이 아웃풋`), 스크립트 실행 지시 등이 포함되어 있었음.

- `하드코드 하지 말라`는 지시도 포함되어 있었음.

- 스킬 개선 작업

- 초기 생성된 리포트 내용이 너무 간단하여, 개선을 요청함.

- 구체적인 내용 (예: 핵심 판단, 상품 인사이트, 채널 인사이트, 액션 플랜, 데이터 한계) 추가를 요청함.

- 코덱스가 스킬을 수정하여 요청한 내용들을 포함한 리포트 템플릿을 생성함.

- 개선된 스킬 테스트 (2026년 4월 데이터 활용)

- 새로운 4월 데이터로 테스트를 진행하고자 함.

- 기존 다운로드 데이터를 사용하지 않고, 매출 대시보드에서 새롭게 CSV를 다운로드받아 사용해달라고 요청함.

- 코덱스 실행 결과: 로컬 대시보드 서버를 띄우고, 새롭게 CSV를 다운로드받아 파일 원본으로 리포트를 생성함.

- 리포트 내용 확인: 핵심 요약, 상품별 매출, 채널별 매출, 매출 증대 액션 플랜 등이 포함됨.

- 파일 경로 확인: 요청한 경로에 정확히 파일이 저장되었음.

- 데이터 정확성 검증: 4월 데이터(97만 5천원, 주문수 12건)와 생성된 리포트의 데이터가 정확히 일치함을 확인함.

- 결론: 웹사이트에서 버튼 클릭을 통한 다운로드, 파일 정리, 특정 템플릿의 리포트 생성 등 복잡했던 자동화 작업이 시현을 통해 손쉽게 구현됨.

## **Record & Replay 활용 예시 2: 네이버 블로그 초안 저장 자동화**

- 자동화 시나리오: AI의 도움을 받아 작성 및 검수된 블로그 글을 네이버 블로그에 초안으로 저장하는 작업.

- 문제점: 네이버 블로그 자동화는 일반적으로 어려움이 있음.

- Record & Replay를 이용한 자동화 과정 (시현)

- Record & Replay 플러그인을 활성화함.

- 작업 내용 프롬프트 설명: "미리 써 둔 마크다운 글을 네이버 블로그 글쓰기 에디터로 옮겨 임시 저장을 하고 싶다. 공개 발행은 하지 않고 초안 저장까지 진행해 줘라"고 요청함.

- 코덱스의 화면 녹화 요청에 'Allow' 버튼을 눌러 녹화를 시작함.

- 실제 작업 시현:

- 네이버 블로그에 접속함.

- '글쓰기' 버튼을 클릭함.

- 블로그 샘플 파일에서 제목을 복사하여 글쓰기 에디터의 제목 입력란에 붙여넣음.

- 블로그 샘플 파일에서 글 내용을 복사하여 본문 입력란에 붙여넣음.

- '저장' 버튼을 클릭하여 초안을 저장함.

- 작업 완료 후 녹화를 중지함.

- 코덱스의 스킬 생성 및 분석

- 시현된 작업을 이해하고 스킬을 생성함.

- 생성된 스킬에서 중요한 점은 네이버 블로그에 이미 로그인된 세션을 유지하면서 작업해야 한다는 점임.

- 생성된 스킬에는 "이미 로그인이 되어 있는 크롬 세션을 활용"한다는 내용이 포함되어 있음을 확인함.

- 만약 이 내용이 없었다면, 스킬 수정 요청을 통해 기존 로그인 세션 활용을 명시하도록 지시해야 함.

- 생성된 스킬 테스트 (두 번째 주간 업무 리포트 관련 블로그)

- 새로운 블로그 글 ("두 번째 주간 업무 리포트") 경로를 복사하여 초안 저장을 요청함.

- 코덱스 실행 결과: 크롬에 새로운 세션을 열어 페이지를 띄움.

- 블로그 작성 탭으로 이동하여 제목과 본문을 입력하고 저장 버튼을 클릭하는 과정을 확인함.

- 작업 완료 후, 실제로 해당 글이 초안으로 저장된 것을 확인함.

- 추가적으로 서식이나 커스텀이 필요한 경우, 스킬 개선 요청을 통해 반영할 수 있음.

- 결론: 네이버 블로그와 같이 자동화가 어려웠던 웹 서비스에서도 시현을 통해 손쉽게 스킬을 생성하고 자동화할 수 있음을 확인함.

## **Record & Replay 기능의 요약 및 활용 팁**

- 핵심 기능: 반복 업무를 글로 설명하는 대신 화면으로 시현하여, 코덱스가 이를 '재사용 가능한 스킬'로 만들어 주는 기능임.

- 활용 이점:

- 기존에 웹에서 업로드, 다운로드 등 자동화하기 어려웠던 작업들을 손쉽게 스킬로 생성하고 자동화할 수 있음.

- 특히 API가 없거나 매크로 작성이 어려운 비개발자에게 유용함.

- 활용 팁:

- 스킬이 처음 생성되면 그대로 활용하기보다는, 먼저 스킬이 어떤 식으로 생성되었는지 확인하는 것이 좋음.

- 보완할 부분이 있다면 추가적으로 피드백을 주어 스킬을 개선한 후 활용하는 것을 권장함.
