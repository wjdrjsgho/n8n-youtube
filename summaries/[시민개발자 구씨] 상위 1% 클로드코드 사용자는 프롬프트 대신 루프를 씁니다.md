## **서론: AI 업무 자동화의 새로운 접근 방식**

- 클로드 코드 활용 시 매번 프롬프트로 업무를 요청하는 방식이 비효율적으로 느껴짐을 설명함.

- 클로드 코드가 여러 스텝의 업무를 한 번에 처리해 주어 편리하나, 익숙해지면 프롬프트 요청 자체도 비효율적으로 느껴지는 경우가 있음을 언급함.

- 클로드 코드 개발자인 보리스 체니가 매번 프롬프트를 입력하는 대신 '루프'를 활용한다고 공개적으로 밝힘.

- 루프는 한 번 잘 설계하여 걸어두면 AI가 다음 일감을 스스로 처리하고, 다음 차례까지 기다려 다시 일을 수행하는 방식임을 설명함.

- 이러한 방식으로 일하는 시스템 구축을 '루프 엔지니어링'이라 표현하기도 함.

- 보리스 체니는 주로 개발 업무에 루프 엔지니어링을 적용하고 있으나, 개발자가 아니더라도 업무 효율화를 위한 적용 영역을 충분히 생각해 볼 수 있음을 언급함.

- 클로드 코드의 루프 기능을 활용하여 반복 업무를 자동화하는 방법을 시연할 예정임을 밝힘.

- 매주 특정 작업을 클로드 코드와 함께 하는 사용자들에게 유용할 것이며, 영상 시청 후 개인 워크플로우에 적용해 볼 것을 권장함.

## **루프 기능 및 관련 기능 비교**

- 실습에 들어가기 전, 혼동될 수 있는 몇 가지 기능들을 비교함.

- <strong>루프 (Loop):</strong>

- 같은 작업을 정해진 간격마다 다시 실행해 줄 수 있는 기능임을 설명함.

- 예시: 30분마다 시트에서 새로운 문의를 처리하는 작업을 반복할 수 있음을 듦.

- 주로 두 가지 유형의 업무에 사용하기 좋음을 설명함.

- 업무가 특정 단계로 진행되어 항상 다음 단계로 넘겨줘야 할 때 작업을 자동화하는 데 유용할 수 있음.

- 동일 업무가 실시간으로 계속 쌓이는 경우, 주기적으로 체크하여 반복 작업을 실행해 주는 데 활용하기 좋음을 언급함.

- 중요한 점: '매주 월요일 아침에 확인해 줘'와 같은 알림용으로는 좋지 않음을 강조함.

- 용도: 이미 진행 중인 일감을 AI가 계속 이어받아 다음 단계로 보내줘야 할 때 활용하는 용도로 적합함을 설명함.

- <strong>슬래시고 (Slashgo):</strong>

- 특정 조건이 될 때까지 작업을 멈추지 않고 진행해 주는 기능임을 설명함.

- 매개가 끝날 때마다 조건이 충족됐는지를 확인하고, 충족이 안 되면 조건이 충족될 때까지 계속 작업을 진행함을 설명함.

- 한 번에 처리해야 할 양이 많은 복잡한 업무를 완수해야 할 때 요청하면 좋음을 언급함.

- <strong>에이전트 크론 (Agent Cron) / 클로드 코드 루틴스 (Claude Code Routines):</strong>

- 정해진 시간에 알아서 일을 하고 보고를 해 주는 AI 직원 같은 기능으로 생각하면 됨.

- 장점: PC가 꺼져 있어도 정해진 스케줄에 독립적으로 할 일을 하고 결과를 보고해 줄 수 있음을 설명함.

- 오늘 다루는 루프와는 결이 다르다고 설명함.

- 결론: 어떤 한 가지 방식이 항상 더 좋기보다는, 세 가지 방식을 각각 적합한 프로젝트에 활용하는 것이 중요함을 강조함.

- 오늘은 그중 첫 번째 기능인 루프에 대해 더 자세히 알아볼 예정임을 밝힘.

## **VS 코드 환경 설정 및 프로젝트 준비**

- VS 코드를 활용하여 실습을 진행할 예정임을 밝힘.

- VS 코드에서 우측 'New Terminal'을 통해 터미널을 염.

- 터미널에서 클로드 코드 설치 커맨드를 입력하여 설치함 (이미 설치되어 있다면 활용함).

- 프로젝트 폴더를 생성함 (예: '루프 샘플' 폴더).

- 생성한 폴더를 'Open Folder'로 열어준 상태임을 설명함.

- 터미널 실행 시 프로젝트 안으로 이미 들어와 있는 상태임을 설명함.

- 이 상태에서 '클로드'를 입력하여 클로드 코드로 진입함.

- Shift + Tab을 눌러 '오토 모드'로 변경함.

## **운영 참고 자료 추가 및 구글 워크스페이스 CLI 연동**

- 루프 셋업 시 참고하여 활용할 몇 가지 운영 문서들을 추가할 예정임을 설명함.

- **`assets` 폴더 생성 및 문서 추가:**

- `assets` 폴더를 추가함.

- 다음 세 가지 문서를 추가함.

- `business_context.md`: 회사 맥락을 담음 (예: '주루 응대'라는 SaaS 회사).

- `product.md`: 트루대 상품의 핵심 기능 및 요금제 설정 정보를 담음.

- `sample_sheet.xlsx`: 고객 피드백과 아웃바운드 리드 두 가지 시트를 생성함.

- 고객 피드백 시트: 서비스에 대한 고객 피드백 내용, 답변 방식, FAQ 추가 여부 판단 정보가 있음.

- 아웃바운드 리드 시트: 아웃바운드 리드를 수집하는 별도의 자동화 시스템을 통해 회사 담당 이메일과 함께 리드들이 자동으로 계속 쌓인다고 가정함. 이 쌓인 메일들에 대해 아웃바운드 메일을 보내는 작업을 자동화할 것임을 설명함.

- **구글 워크스페이스 CLI 설치 및 연동 요청:**

- 엑셀 시트 정보를 구글 시트로 그대로 옮겨 구글 시트에서 작업하고 싶다고 가정함.

- 메일을 보내려면 Gmail과 연동하여 메일 초안을 작성할 수 있어야 함.

- 이러한 구글 프로덕트들과 연동할 수 있도록 구글 워크스페이스 CLI 설치 및 연동을 요청함.

- 해당 프로젝트에 구글 워크스페이스 CLI가 설치되어 있는지 체크하고, 없으면 설치를 요청하는 방식으로 진행함.

- 깃허브 링크까지 제공함.

- 다른 프로젝트에서 이미 사용 중이므로 설치된 것을 찾아 셋업을 진행하며, 처음 설치 시에는 클로드 코드가 자세한 설정 방법을 설명해 줄 것임을 언급함. 가이드에 따라 설정하면 구글 계정과 연동하여 작업 진행이 가능함.

- 셋업 완료 후, 엑셀 시트를 구글 시트로 변환해 달라고 요청하고, 변환 완료 후 링크를 통해 생성된 시트를 확인함.

## **루프 관리 시스템 구축 및 운영 문서 생성**

- 루프를 효율적으로 돌리기 위해 몇 가지 관리 문서들을 생성할 예정임을 설명함.

- **세 가지 운영서 설정:**

- `claude.md`: 기본적인 업무 프로세스를 정의하는 문서로 생성함.

- `loop.md`: 생성할 루프의 구체적인 프롬프트까지 포함하여 '장부 같은 문서'로 별도 생성함.

- `loop_report.md`: 루프가 돌아갈 때마다 제대로 실행되었는지, 언제 어떤 루프가 실행되었는지 그 이력을 담는 문서를 생성함.

- **`loop.md` (루프 장부)를 별도로 만드는 이유:**

- 오늘은 샘플로 두 개의 루프를 돌려 볼 예정이지만, 실제 업무에서는 4개나 5개의 루프를 생성하여 활용할 수도 있음을 언급함.

- 루프는 기본적으로 7일이 지나면 비활성화됨.

- 비활성화된 루프를 다시 실행해야 할 때마다 일일이 프롬프트를 입력하는 것이 비효율적이므로, 매번 돌릴 루프가 있다면 별도의 장부에 추가해 놓고 이 문서를 참고하여 항상 돌리도록 하기 위함임을 설명함.

- **루프 실행 과정:**

- `loop.md`에 추가된 루프들을 하나의 세션에서 문서를 참고하여 돌리게 됨.

- 루프가 실행한 작업(예: 시트 업데이트, 메일 초안 작성)은 시트에 자동으로 업데이트되거나 메일 초안 작성이 실제로 되도록 설정할 것임을 설명함.

- **운영 문서 생성 요청:**

- 클로드 코드에게 `claude.md` (프로젝트의 작업 규칙), `loop.md` (루프 장부), `weekly_loop_report.md` (매주 루프 실행 이력을 관리하는 문서) 세 가지 셀업을 요청함.

- 제공된 샘플 시트(고객 피드백, 아웃바운드 리드)의 컨텍스트 기반으로 '고객 피드백 응대 자동화 루프'와 '아웃바운드 후속 초안 메일 작성 루프' 두 가지를 자동으로 인식하고 작성하는 것을 확인함.

- 클로드 코드의 응답을 통해 `claude.md`에 작업 규칙이, `loop.md`에 데모 컨텍스트에 맞는 두 루프가 생성되어 장부에 추가되었음을 확인하며, `weekly_loop_report.md`도 생성되었음을 확인함.

## **루프 정의 및 주기 설정**

- 생성된 운영 문서들을 확인하고 루프의 주기 설정을 조정함.

- **`claude.md` 내용 확인:**

- `claude.md`에 루프 장부에 대한 설명, 회사 제품 맥락 확인 방법, 매 실행 후 주간 리포트에 한 줄 기록하는 것 등이 잘 기록되어 있음을 확인함.

- **`loop.md` 내용 확인 및 주기 수정:**

- `loop.md`에 루프별 내용들이 추가되어 있음을 확인함.

- '고객 피드백 분류 및 초안 작성 루프'와 프롬프트 원문이 생성되어 있음을 확인하며, 실제 작업 시 원하는 형태로 수정 가능함을 언급함. 현재는 적절하다고 가정하고 진행함.

- 루프 주기를 수정함.

- 고객 피드백 루프: 주기를 '매일 1회'에서 '1시간 단위'로 수정 요청함 (고객 피드백이 더 자주 쌓인다고 가정).

- 아웃바운드 리드 루프: Gmail 초안 생성 주기를 '30분'으로 설정 요청함 (아웃바운드 리드를 실시간으로 수집한다고 가정할 때 더 빨리 체크하고 빈 것을 보내기 위함).

- 클로드 코드에게 주기 설정을 수정해 달라고 요청하고, 수정 완료를 확인함.

- 주기는 업무 특성에 맞게 자율적으로 설정하면 됨을 설명함.

- 매주 실행하는 정기적인 리포트 형태의 업무는 루프보다는 클로드 루틴이나 허미스 크론을 활용하는 것이 효율적일 수 있음을 다시 추천함.

- 오늘 활용하는 루프는 30분 또는 1시간 단위로 내가 업무를 하는 과정에서 자동으로 체크하고 업무 진척을 챙겨줬으면 하는 업무에 적용하기 좋음을 설명함.

- **`weekly_loop_report.md` 내용 확인:**

- 주간 루프 실행 리포트에 실행 이력을 담을 수 있는 간단한 문서가 생성되었음을 확인함.

## **루프 실행 및 자동 등록**

- 정의된 루프들을 실행하고, 자동 실행으로 등록하는 과정을 설명함.

- **루프 실행 요청:**

- `loop.md` 문서를 클로드 코드가 읽고 상태가 '액티브'인 활성 루프들만 돌려달라고 요청함.

- 이렇게 하면 여러 루프가 생성되어 있을 때도 활성화된 루프들만 클로드 코드가 직접 읽어 알아서 여러 개를 한 번에 셋업할 수 있음을 설명함.

- 실행 결과: 시트에 답변 초안들이 생성되고, FAQ 후보 여부도 체크되는 등 시트에 채워야 할 정보들이 한 번에 채워진 것을 확인함. 아웃바운드 팔로업 작업도 진행되어 Gmail 임시 보관함에 초안들이 생성된 것을 확인함.

- **루프 자동 실행 등록:**

- 방금 실행은 작업만 진행되었고 자동 실행 등록은 안 된 상태임을 설명함.

- `/loop` 명령어로 자동 실행 등록을 요청함.

- 애초에 처음에 한 번에 `/loop`로 자동 실행되게 셋업해 달라고 요청할 수도 있음을 언급함 (이번에는 먼저 제대로 실행되는지 테스트 후 등록 요청).

- 등록 진행 결과: 아웃바운드 팔로업은 30분 단위, 피드백 트리아지는 60분 단위로 실행되게 셋업된 것을 확인함.

- `loop_report.md`에도 루프가 한 번 돌아갈 때마다 실행 이력이 쌓이는 것을 확인하며, 언제, 어떤 루프, 어떤 작업을 했는지 간단히 확인 가능함을 설명함.

## **추가 데이터 처리 및 루프의 효율성 입증**

- 실제로 처리해야 할 데이터들이 시트에 쌓일 때 루프가 어떻게 작동하는지 시연함.

- **시트에 추가 데이터 요청:**

- 시트에 추가 데이터를 요청하여, 루프가 시간이 되면 주기에 도달했을 때 매번 쌓이는 행들을 바로바로 처리해 줄 수 있음을 보여주기 위함임을 설명함.

- 추가된 데이터들을 확인함.

- **루프 수동 실행 요청:**

- 30분, 1시간 주기를 기다릴 수 없으므로 루프 실행을 요청함.
